## 11.1 HomePage — 公开首页

### 基本信息

| 属性 | 值 |
|------|-----|
| 路由 | `/` |
| 文件 | `src/pages/HomePage.jsx` |
| 守卫 | 无，所有角色可访问 |
| 懒加载 | 是 |

### 页面布局

```
Hero 区域
  ├── 搜索框（SearchBar 组件）
  ├── 热门标签快捷入口（最多 6 个 TagBadge）
  └── 快速专题入口（最多 2 个）

编辑推荐区（editors_picks，最多 4 篇）
  └── 每篇带封面图、标题、摘要、栏目标签

体验能力卡片区（experienceCards，5 张卡片）
  ├── 统一搜索
  ├── AI 助理（或升级提示）
  ├── 专题阅读（或升级提示）
  ├── 时光机
  └── 会员访问

角色引导面板（rolePanel）
  └── 根据当前角色（admin/paid_member/free_member/guest）显示不同引导
  
产品能力卡片（productCards，3 张）
  ├── 教学与研究
  ├── 管理者洞察
  └── 会员阅读

AI 助理 CTA
  └── 已有权限 → "进入 AI 助理" 链接到 /chat
  └── 无权限 → "解锁 AI 助理" 链接到 /membership 或 /login

最新文章（latestArticles，初始 12 篇，可加载更多）
  └── ArticleCard 网格排列
  └── "加载更多" 按钮（无限滚动翻页）
```

### 核心交互

1. **搜索提交**：调用 `navigate(/search?q=...&mode=...)`，搜索模式根据角色自动选择（付费/管理员 → smart，其他 → exact）
2. **加载更多文章**：调用 `fetchLatestArticles(12, offset, language)`，将返回结果追加到 `latestArticles` 数组；不足 12 条时隐藏"加载更多"按钮
3. **体验能力卡片动态化**：AI 助理和专题卡片根据当前权限显示不同文案——有权限时显示功能说明和入口链接；无权限时显示升级引导
4. **角色面板差异化**：guest → 介绍登录价值；free_member → 引导升级；paid_member → 突出权益；admin → 引导进入管理台

### 调用的 API

| API 函数 | 端点 | 触发时机 |
|----------|------|----------|
| `fetchHomeFeed(language)` | GET `/api/home/feed` | 组件挂载时、语言切换时 |
| `fetchLatestArticles(12, offset, language)` | GET `/api/articles/latest` | 点击"加载更多"时 |

### 数据来源（fetchHomeFeed 返回值）

| 字段 | 说明 |
|------|------|
| `hot_tags` | 热门标签数组 |
| `topics` | 推荐专题数组 |
| `editors_picks` | 编辑推荐文章 |
| `latest` | 最新文章（初始批次） |

## 11.2 ArticlePage — 文章详情

### 基本信息

| 属性 | 值 |
|------|-----|
| 路由 | `/article/:id` |
| 文件 | `src/pages/ArticlePage.jsx` |
| 守卫 | 无（后端按 access_level 控制正文可见范围） |
| 懒加载 | 是 |

### 页面布局

```
文章元信息区
  ├── 访问层级标签（公开/会员/付费）
  ├── 会员状态标签
  ├── 标题（font-serif，超大字号）
  ├── 副标题
  ├── 作者 + 机构（链接到 /organization/:slug）
  ├── 发布日期
  └── 标签组（TagBadge 组件）

封面图（如果存在）

付费墙提示（如果 access.locked === true）
  └── 锁定提示文字
  └── 升级链接（→ /membership 或 → /login）

正文区域
  ├── 如果有 html_content → AutoHeightPreviewFrame 渲染 HTML
  └── 如果只有 markdown → 内置 Markdown 渲染器将 markdown 转 HTML

操作工具条
  ├── 点赞按钮（Heart 图标，带计数）
  ├── 收藏按钮（Bookmark 图标，带计数）
  ├── 浏览量（Eye 图标）
  └── 翻译按钮（Languages 图标）

AI 摘要面板（可展开/收起）
  └── 调用 fetchArticleSummary 获取，Markdown 渲染

翻译面板（可展开/收起）
  └── 调用 fetchArticleTranslation 获取英文翻译

相关文章推荐（related_articles）
  └── ArticleCard 网格
```

### 核心交互

1. **文章加载**：根据 URL 参数 `:id` 调用 `fetchArticle(id, authToken)` 获取文章数据
2. **互动数据加载**：调用 `fetchArticleEngagement(id, authToken)` 获取当前用户的点赞/收藏状态和计数
3. **点赞/收藏**：调用 `submitArticleReaction(id, { type: 'like'|'bookmark', active: true|false }, authToken)` 切换状态
4. **AI 摘要**：点击"AI 摘要"按钮调用 `fetchArticleSummary(id, authToken)`，首次加载后缓存
5. **文章翻译**：点击"翻译"按钮调用 `fetchArticleTranslation(id, 'en', authToken)`，首次加载后缓存
6. **Markdown 渲染器**：内置简易 Markdown → HTML 转换器，支持标题（h1-h6）、段落、无序列表、有序列表、水平线、加粗、斜体、行内代码
7. **HTML 正文渲染**：使用 `AutoHeightPreviewFrame` 组件在沙盒 iframe 中渲染 HTML，自动适配高度
8. **付费墙逻辑**：后端在 `access` 字段中返回 `locked: true` 和 `access_level`（'member'/'paid'）；前端根据此信息显示锁定提示，并引导用户登录或升级

### 调用的 API

| API 函数 | 端点 | 触发时机 |
|----------|------|----------|
| `fetchArticle(id, authToken)` | GET `/api/article/:id` | 页面加载、id 变化 |
| `fetchArticleEngagement(id, authToken)` | GET `/api/article/:id/engagement` | 页面加载 |
| `submitArticleReaction(id, payload, authToken)` | POST `/api/article/:id/reaction` | 点赞/收藏操作 |
| `fetchArticleSummary(id, authToken)` | GET `/api/summarize_article/:id` | 点击 AI 摘要 |
| `fetchArticleTranslation(id, lang, authToken)` | GET `/api/article/:id/translation` | 点击翻译 |

### 访问控制映射

| 文章 access_level | guest | free_member | paid_member | admin |
|-------------------|-------|-------------|-------------|-------|
| public | 全文 | 全文 | 全文 | 全文 |
| member | 预览 | 全文 | 全文 | 全文 |
| paid | 预览 | 预览 | 全文 | 全文 |

## 11.3 SearchPage — 统一搜索

### 基本信息

| 属性 | 值 |
|------|-----|
| 路由 | `/search` |
| 文件 | `src/pages/SearchPage.jsx` |
| 守卫 | 无 |
| 懒加载 | 是 |

### 查询参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `q` | 搜索关键词 | `''` |
| `mode` | 搜索模式（`smart` / `exact`） | `'smart'` |
| `sort` | 排序（`relevance` / `date_desc` / `date_asc`） | `'relevance'` |
| `start_date` | 起始日期过滤 | `''` |
| `end_date` | 结束日期过滤 | `''` |
| `tags` | 标签 slug 列表（逗号分隔） | `''` |
| `columns` | 栏目 slug 列表（逗号分隔） | `''` |

### 页面布局

```
SearchBar 组件（带模式切换）

搜索历史（无关键词时显示，最多 8 条，localStorage 持久化）

智能搜索权限提示（仅无权限时显示）

筛选器面板
  ├── 排序切换（相关性 / 最新 / 最早）
  ├── 日期范围筛选器（起止日期输入）
  ├── 标签多选筛选器（从 fetchTags 获取全部标签组）
  └── 栏目多选筛选器（从 fetchColumns 获取全部栏目）

搜索结果区
  ├── 结果计数 + 搜索耗时
  ├── 当前激活的过滤条件标签（可单独移除）
  ├── AI 智能摘要（仅 smart 模式返回）
  └── ArticleCard 结果列表
```

### 核心交互

1. **搜索执行**：SearchBar 提交后更新 URL 查询参数 → `useEffect` 监听参数变化 → 调用 `searchArticles(payload, authToken)`
2. **搜索模式自动降级**：如果当前用户无 smart 搜索权限且 URL 中 mode=smart，自动替换为 mode=exact 并更新 URL
3. **搜索历史**：每次搜索将关键词写入 localStorage（key: `fdsm-search-history`），最多保留 8 条，去重，最新在前
4. **筛选器联动**：修改任何筛选条件时重新构建 URL 并导航，触发新搜索
5. **标签/栏目多选**：点击标签/栏目 toggle 其选中状态，已选标签以 TagBadge 形式显示在结果区上方，可点击 X 移除

### 调用的 API

| API 函数 | 端点 | 触发时机 |
|----------|------|----------|
| `searchArticles(payload, authToken)` | POST `/api/search` | 查询参数变化时 |
| `fetchTags()` | GET `/api/tags` | 组件挂载时 |
| `fetchColumns()` | GET `/api/columns` | 组件挂载时 |

### 搜索请求体

```json
{
  "query": "AI 战略",
  "mode": "smart",
  "language": "zh",
  "sort": "relevance",
  "page": 1,
  "page_size": 18,
  "filters": {
    "start_date": "2024-01-01",
    "end_date": null,
    "tags": ["人工智能"],
    "columns": ["insights"]
  }
}
```

### 搜索模式对比

| 特性 | exact（精确搜索） | smart（智能搜索） |
|------|-------------------|-------------------|
| 权限 | 所有用户 | 付费会员 + 管理员 |
| 底层 | BM25 关键词匹配 | 向量检索 + 语义理解 |
| 返回 | 文章列表 | 文章列表 + AI 摘要 |
| 建议词 | 支持 | 支持 |

## 11.4 ChatPage — AI 助理

### 基本信息

| 属性 | 值 |
|------|-----|
| 路由 | `/chat` |
| 文件 | `src/pages/ChatPage.jsx` |
| 守卫 | `requireAiAssistant`，fallbackPath `/membership` |
| 懒加载 | 是 |

### 访问权限

- 付费会员（`isPaidMember`）和管理员（`isAdmin`）可访问
- 未登录用户 → 重定向到 `/login?redirect=/chat`
- 已登录但非付费/管理员 → 重定向到 `/membership`

### 页面布局

```
页面标题区
  ├── Section kicker: "AI 助理"
  └── 标题: "围绕知识库继续提问、比较与整合"

ChatPanel（variant="page" 模式）
  ├── 会话侧边栏（仅 page 模式显示）
  │   ├── "新建会话" 按钮
  │   ├── 历史会话列表（可点击加载、可删除）
  │   └── 空状态提示
  │
  ├── 对话主区域
  │   ├── 问候消息（含快捷指令介绍）
  │   ├── 快捷指令按钮组（6 个）
  │   ├── 对话消息流
  │   │   ├── 用户消息
  │   │   └── AI 回复（ReactMarkdown 渲染）
  │   │       ├── 引用来源（可点击链接到文章）
  │   │       └── 后续操作建议（可点击自动填入）
  │   └── 输入框 + 发送按钮
  │
  └── Loading 状态指示
```

### 核心交互

1. **发送消息**：用户输入后调用 `sendChatMessage({ message, session_id, language })`
2. **快捷指令**：点击快捷按钮将预设文本填入输入框并自动发送
3. **会话管理**：
   - 列表：`fetchChatSessions()` 获取所有会话
   - 加载：`fetchChatSession(sessionId)` 获取会话历史
   - 删除：`deleteChatSession(sessionId)` 删除指定会话
   - 新建：清空当前消息和 sessionId
4. **后续操作**：AI 回复可附带 `followUps` 数组，渲染为可点击按钮，点击后自动填入输入框发送

### 快捷指令列表

| 中文指令 | 英文指令 | 功能 |
|----------|----------|------|
| `/今日一读` | `/today` | 获取今日推荐阅读 |
| `/简报 AI战略` | `/brief AI strategy` | 生成主题简报 |
| `/总结 ESG转型` | `/summarize ESG transition` | 总结主题内容 |
| `/比较 AI 与 ESG` | `/compare AI vs ESG` | 对比两个主题 |
| `/时间线 生成式AI` | `/timeline Generative AI` | 生成时间线 |
| `/继续阅读` | `/recommend` | 获取阅读推荐 |

### 调用的 API

| API 函数 | 端点 | 触发时机 |
|----------|------|----------|
| `sendChatMessage(payload)` | POST `/api/chat` | 发送消息时 |
| `fetchChatSessions()` | GET `/api/chat/sessions` | 组件挂载时、sessionId 变化时 |
| `fetchChatSession(id)` | GET `/api/chat/session/:id` | 点击历史会话时 |
| `deleteChatSession(id)` | DELETE `/api/chat/session/:id` | 点击删除会话时 |

### 页面布局

```
Hero 区域（深蓝渐变背景）
  ├── 标题："上传、改稿、自动排版、预览、发布"
  ├── 说明文字
  ├── "新建草稿" 按钮
  ├── "上传文稿" 按钮（支持 .md .txt .html .htm .docx）
  └── Dashboard 统计卡片（3 张）
      ├── 草稿数
      ├── 待审核数
      └── 已发布数

主内容区（两栏布局）
  ├── 左侧（0.72fr）
  │   ├── 稿件列表面板
  │   │   └── 按钮列表，每项显示标题和工作流状态
  │   │   └── 选中项高亮
  │   └── 流程控制面板
  │       ├── 审核备注 textarea
  │       ├── 定时发布 datetime-local 输入
  │       ├── "提交审核" 按钮
  │       └── "发布" 按钮
  │
  └── 右侧（1.28fr）
      ├── 操作按钮栏
      │   ├── 保存
      │   ├── 一键自由排版（AI，使用 gemini-3-flash-preview）
      │   ├── 自动标签
      │   └── 生成预览
      │
      ├── 表单字段区（4 列网格）
      │   ├── 标题、副标题
      │   ├── 作者、机构
      │   ├── 发布日期、所属栏目（下拉选择）
      │   ├── 文章类型、主题
      │   ├── 访问层级（public/member/paid）
      │   ├── 排版模式（auto/manual）
      │   ├── 排版备注
      │   ├── 来源 URL、封面图 URL
      │   └── 源稿 Markdown + 正文 Markdown（双 textarea，带字数统计）
      │
      ├── 预览切换（Web 版 / 公众号版）
      └── HTML 预览（AutoHeightPreviewFrame）
          └── 导出链接（Web 版 / 公众号版 HTML 文件下载）
```

### 核心交互

1. **文稿上传**：选择文件 → `uploadEditorialFile(file)` → 后端解析并创建文章 → 刷新列表
2. **新建草稿**：清空所有表单字段和选中 ID
3. **保存**：调用 `persist()`（create 或 update） → 刷新列表
4. **一键自由排版**：先保存 → 调用 `autoFormatEditorialArticle(id, { source_markdown, layout_mode, formatting_notes })` → 后端使用 Gemini 模型排版 → 刷新
5. **自动标签**：先保存 → 调用 `autotagEditorialArticle(id)` → 后端 AI 生成标签 → 刷新
6. **生成预览**：先保存 → 调用 `renderEditorialHtml(id)` → 后端生成 HTML → 刷新预览区
7. **提交审核**：先保存 → 调用 `updateEditorialWorkflow(id, { action: 'submit_review', review_note, ... })`
8. **发布**：先保存 → 调用 `publishEditorialArticle(id)` → 文章状态变为已发布
9. **稿件切换**：点击列表中的稿件 → `syncDetail(id)` 加载详情并填充表单

### 调用的 API

| API 函数 | 端点 | 触发时机 |
|----------|------|----------|
| `fetchEditorialArticles(80)` | GET `/api/editorial/articles` | 初始化、每次操作后刷新 |
| `fetchEditorialDashboard(8)` | GET `/api/editorial/dashboard` | 初始化、每次操作后刷新 |
| `fetchEditorialArticle(id)` | GET `/api/editorial/articles/:id` | 选中稿件时 |
| `fetchColumns()` | GET `/api/columns` | 初始化 |
| `createEditorialArticle(payload)` | POST `/api/editorial/articles` | 新建保存 |
| `updateEditorialArticle(id, payload)` | PUT `/api/editorial/articles/:id` | 更新保存 |
| `autoFormatEditorialArticle(id, payload)` | POST `/api/editorial/articles/:id/auto-format` | 一键排版 |
| `autotagEditorialArticle(id)` | POST `/api/editorial/articles/:id/autotag` | 自动标签 |
| `renderEditorialHtml(id)` | POST `/api/editorial/articles/:id/render-html` | 生成预览 |

## 11.25 全局交互模式

### 11.25.1 公共数据加载模式

所有页面采用一致的数据加载模式：

```jsx
const [data, setData] = useState(null)
const [error, setError] = useState('')

useEffect(() => {
  fetchXxx()
    .then(setData)
    .catch(() => setError('加载失败'))
}, [dependencies])
```

部分页面增加了 cleanup 函数防止 unmount 后的状态更新：

```jsx
useEffect(() => {
  let mounted = true
  fetchXxx().then((payload) => {
    if (!mounted) return
    setData(payload)
  })
  return () => { mounted = false }
}, [])
```

### 11.25.2 分页加载模式

多个列表页面（Column、Tag、Topic、Organization、HomePage latest）使用相同的"加载更多"模式：

```jsx
const handleLoadMore = async () => {
  setLoadingMore(true)
  try {
    const nextPage = data.page + 1
    const payload = await fetchXxx(slug, nextPage, PAGE_SIZE)
    setData((current) => ({
      ...payload,
      items: [...(current?.items || []), ...payload.items],
    }))
  } finally {
    setLoadingMore(false)
  }
}
```

### 11.25.3 关注模式

Column、Tag、Topic 三类实体共享相同的关注交互：

1. `fetchFollows(accessToken)` 检查当前用户的关注列表
2. `some()` 判断是否已关注当前实体
3. `toggleFollow({ entity_type, entity_slug, active })` 切换关注状态
4. 未登录时点击关注 → `openAuthDialog()` 弹出登录对话框

### 11.25.4 多语言文案模式

所有页面使用两种国际化策略：

1. **messages.js 翻译**：通过 `t('key')` 获取预定义翻译（用于固定 UI 文案）
2. **行内 byLanguage**：`isEnglish ? 'English text' : '中文文字'`（用于动态/页面级文案）

### 11.25.5 错误处理模式

```jsx
{error ? <div className="mt-6 text-sm text-red-500">{error}</div> : null}
```

错误信息统一以红色文字显示在页面主内容区上方。部分页面（如 AdminMembershipsPage）根据 HTTP 状态码（401、403）显示差异化提示。

### 11.25.6 Loading 状态模式

- **全局 Suspense fallback**：`PageFallback` 组件显示"加载中"文字
- **页面级 loading**：各页面自行管理 loading state
- **操作级 busy**：EditorialWorkbenchPage 使用 `busy` 字符串标识当前正在执行的操作，对应按钮显示 `LoaderCircle` 旋转图标

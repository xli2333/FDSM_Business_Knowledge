# 07 --- AI 对话系统

> 本章完整拆解 `backend/routers/chat.py` 中的对话路由、六大斜杠命令、自由问答 RAG 流程、会话持久化机制，以及置信度计算规则。

## 7.1 端点规格

### 7.1.1 POST `/api/chat` --- 主对话入口

**请求体 `ChatRequest`：**

```python
class ChatMessageIn(BaseModel):
    role: str            # "user" 或 "assistant"
    content: str         # 消息文本

class ChatRequest(BaseModel):
    messages: list[ChatMessageIn]   # 对话消息列表
    session_id: str | None = None   # 会话 ID，首次为 None 时自动生成
    mode: str = "precise"           # 搜索模式（预留，当前未使用）
    language: str = "zh"            # 响应语言，"zh" 或 "en"
```

**响应体 `ChatResponse`：**

```python
class ChatResponse(BaseModel):
    session_id: str                                     # 会话 ID
    answer: str                                         # Markdown 格式的回答
    sources: list[ArticleCard] = Field(default_factory=list)   # 引用的文章卡片
    follow_up_questions: list[str] = Field(default_factory=list)  # 后续建议问题（最多 3 条）
    confidence: float = 0.0                             # 置信度 0.0-1.0
```

### 7.1.2 辅助端点

| 方法 | 路径 | 响应模型 | 说明 |
|------|------|---------|------|
| GET | `/api/chat/sessions` | `list[ChatSessionSummary]` | 列出最近 30 个会话 |
| GET | `/api/chat/session/{session_id}` | `ChatSessionDetail` | 获取单个会话的完整消息记录 |
| DELETE | `/api/chat/session/{session_id}` | `ChatSessionDeleteResponse` | 删除会话及其所有消息 |

## 7.2 请求处理主流程

`POST /api/chat` 的 `chat()` 函数按以下步骤执行：

```
1. 提取最近一条 user 消息作为 latest_question
2. 语言规范化 _normalize_language（CJK 检测强制 zh）
3. 会话 ID：取 request.session_id，为 None 则 uuid4().hex 新建
4. 截取最近 6 条消息拼为 history_text
5. 尝试 _dispatch_command(latest_question, ...)
   ├── 命中命令 → 走命令流程
   └── 未命中 → 走自由问答 RAG 流程
6. 持久化：store_chat_session + append_chat_message（user + assistant）
7. 返回 ChatResponse，follow_up_questions 截断到 3 条
```

**语言规范化细节：** `_normalize_language(requested_language, text)` 的判断优先级为：
1. 若 `text` 包含 CJK 字符 → 强制 `"zh"`。
2. 否则，若 `requested_language` 显式为 `"en"` → `"en"`。
3. 其余情况 → `"zh"`。


## 7.4 六大命令详解

### 7.4.1 `/brief [topic]` --- 战略简报

**别名：** `/简报`

**参数：** `topic`（必填），缺失时返回用法提示。

**执行逻辑：**
1. 调用 `rag_engine.search_articles(topic, mode="smart", page=1, page_size=6)`，检索 **6 篇**文章。
2. 构建专用 Prompt 问题：`"Create an executive brief on '{topic}'. Cover the core signals, business implications, what deserves immediate attention, and the next questions worth tracking."`
3. 调用 `_answer_from_sources` -> `ai_service.answer_with_sources` 生成回答。
4. 置信度：有源文章时 **0.88**，无源时 0.3。

**后续建议：** `/timeline {topic}`、`/summarize {topic}`、`/recommend {topic}`

---

### 7.4.2 `/summarize [topic]` --- 主题总结

**别名：** `/总结`、`/摘要`

**参数：** `topic`（必填），缺失时返回用法提示。

**执行逻辑：**
1. 调用 `rag_engine.search_articles(topic, mode="smart", page=1, page_size=6)`，检索 **6 篇**文章。
2. 构建专用 Prompt 问题：`"Summarize the core viewpoints, representative themes, and open questions around '{topic}'. Keep the answer dense and source-grounded."`
3. 调用 `_answer_from_sources` -> `ai_service.answer_with_sources` 生成回答。
4. 置信度：有源文章时 **0.85**，无源时 0.3。

**后续建议：** `/brief {topic}`、`/timeline {topic}`、`/recommend {topic}`

---

### 7.4.3 `/timeline [topic]` --- 时间线梳理

**别名：** `/时间线`、`/脉络`

**参数：** `topic`（必填），缺失时返回用法提示。

**执行逻辑：**
1. 调用 `rag_engine.search_articles(topic, mode="smart", sort="date", page=1, page_size=8)`，检索 **8 篇**文章，按 **日期排序**。
2. 构建专用 Prompt 问题：`"Explain the evolution of '{topic}' as a timeline. Highlight key dates, turning points, and how the framing changed over time."`
3. 调用 `_answer_from_sources` -> `ai_service.answer_with_sources` 生成回答。
4. 置信度：有源文章时 **0.83**，无源时 0.3。

**后续建议：** `/brief {topic}`、`/summarize {topic}`、`/recommend {topic}`

---

### 7.4.4 `/compare [A] vs [B]` --- 差异对比

**别名：** `/比较`、`/对比`

**参数：** 两个主题，用 `vs`、`versus`、`与`、`和` 分隔（正则 `\s+(?:vs\.?|versus|与|和)\s+`，不区分大小写）。格式不正确时返回用法提示。

**执行逻辑：**
1. 分别检索两个主题：
   - `rag_engine.search_articles(left, mode="smart", page=1, page_size=4)` -> **4 篇**
   - `rag_engine.search_articles(right, mode="smart", page=1, page_size=4)` -> **4 篇**
2. 调用 `_dedupe_sources(left_sources, right_sources)` 按文章 ID 去重合并（最多 8 篇，实际可能更少）。
3. 构建专用 Prompt 问题：`"Compare '{left}' and '{right}'. Show the overlap, the major differences, and what a business reader should read next."`
4. 调用 `_answer_from_sources` -> `ai_service.answer_with_sources` 生成回答。
5. 置信度：有源文章时 **0.84**，无源时 0.3。

**后续建议：** `/timeline {left}`、`/timeline {right}`、`/brief {left}`

---

### 7.4.5 `/today [YYYY-MM-DD]` --- 今日一读

**别名：** `/今日一读`、`/一读`、`/今日`、`/今天`、`/今日简报`

**参数：** 可选的 ISO 日期字符串 `YYYY-MM-DD`。不传则使用当天日期。格式错误（非空且不匹配 `\d{4}-\d{2}-\d{2}`）时返回用法提示。

**执行逻辑：**
1. 调用 `catalog_service.get_daily_read(target_date, language)` 获取每日推荐文章。
2. **选文算法细节：**
   - 从数据库取不超过 `anchor_date` 的最近 240 篇文章。
   - 过滤非商业安全文章（排除 editorial、test、demo 等标记）。
   - 筛选近 **180 天**内的候选文章，无合格则取前 96 篇。
   - 按 `_daily_read_priority` 综合评分排序取 TOP 12：
     - `freshness = max(0, 180 - age_days) / 18` --- 越新越高，最高 10 分
     - `popularity = min(view_count / 1800, 3.4)` --- 阅读量加成，上限 3.4
     - `feature_boost = 2.2 if is_featured else 0.0` --- 编辑推荐加成
     - `depth_boost = min(word_count / 2600, 1.4)` --- 长文加成
   - **SHA1 轮转选择**：`hashlib.sha1(anchor_date.isoformat()).hexdigest()[:8]` 转为整数对候选列表长度取模，同一日期永远选同一篇。
3. 调用 `_build_daily_read_answer` 构建结构化回答，包含标题、发布日期、引语、主线提示、选文说明。
4. 置信度：有源文章时 **0.88**，无源时 0.45。

**后续建议：** 基于选中文章主题动态生成 `/brief`、`/timeline`、`/recommend`。

---

### 7.4.6 `/recommend [subject]` --- 继续阅读 / 延伸推荐

**别名：** `/继续阅读`、`/延伸阅读`、`/推荐`

**参数：** 可选的主题关键词 `subject`。

**执行逻辑（两条路径）：**

**路径 A --- 有 subject 参数（标签关联推荐）：**
1. 调用 `rag_engine.search_articles(subject, mode="smart", page=1, page_size=5)` 检索 **5 篇**。
2. `recommendation_mode = "topic"`。

**路径 B --- 无 subject 参数（高信号文章推荐）：**
1. 调用 `catalog_service.get_recommended_articles(session_id, limit=5)` 获取推荐。
2. 内部逻辑：
   - 先通过 `_load_context_source_ids` 从当前会话的最近 8 条 assistant 消息中提取已引用文章 ID。
   - 若有上下文文章 ID，基于标签关联查找相关文章（`contextual` 模式）。
   - 若无上下文，退回全站高信号文章列表。
3. `recommendation_mode` 由 `get_recommended_articles` 返回（`"contextual"` 或 `"high_signal"`）。

**通用后处理：**
1. 调用 `_build_continue_reading_answer` 生成带有"为什么读"标注的阅读列表。
2. 置信度：有源文章时 **0.8**，无源时 0.35。

**后续建议：** `/today`、`/brief {topic}`、`/timeline {topic}`

## 7.5 自由问答流程（RAG）

当用户输入不以 `/` 开头时，进入自由问答流程：

```
用户问题 latest_question
        |
        v
rag_engine.search_articles(
    latest_question,
    mode="smart",
    page=1,
    page_size=5,       # 检索 5 篇
    language=response_language,
)
        |
        v
sources = search_payload["items"]
        |
        v
_answer_from_sources(
    latest_question,
    history_text,        # 最近 6 条消息拼接
    sources,
    latest_question,     # fallback_question
    response_language,
)
        |
        ├── AI 可用且有源 → ai_service.answer_with_sources(...)
        |       构建 context_blocks：每篇格式为
        |       "[i] Title: ... / Date: ... / Topic: ... / Excerpt: ..."
        |
        └── AI 不可用或异常 → _fallback_answer(...)
                返回阅读路径建议列表
        |
        v
follow_up_questions = _build_follow_ups(latest_question, sources, language)
        |
        v
confidence = min(1.0, sources[0]["score"] / 12) if sources else 0.45
```

**关键参数差异（与命令对比）：**

| 维度 | 自由问答 | 命令 |
|------|---------|------|
| 检索数量 | 5 篇 | 6/8/4+4 篇不等 |
| 搜索模式 | `smart` | `smart` |
| 排序 | 默认（相关性） | `/timeline` 用 `date` |
| Prompt 问题 | 用户原文 | 预设的结构化指令 |
| 置信度 | 动态计算 | 固定值 |

## 7.6 会话持久化

### 7.6.1 数据表结构

**`chat_sessions` 表：**

| 列名 | 类型 | 说明 |
|------|------|------|
| `id` | TEXT PK | 会话 ID（`uuid4().hex` 或客户端传入） |
| `title` | TEXT | 会话标题（取用户首条问题前 40 字符） |
| `created_at` | TEXT (ISO 8601) | 创建时间 |
| `updated_at` | TEXT (ISO 8601) | 最后更新时间 |
| `last_question` | TEXT | 最近一次用户问题 |

**`chat_messages` 表：**

| 列名 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK AUTOINCREMENT | 自增主键（隐含） |
| `session_id` | TEXT FK | 关联的会话 ID |
| `role` | TEXT | `"user"` 或 `"assistant"` |
| `content` | TEXT | 消息内容 |
| `sources_json` | TEXT | JSON 序列化的 `list[dict]`，记录引用文章 |
| `follow_ups_json` | TEXT | JSON 序列化的 `list[str]`，记录后续建议 |
| `created_at` | TEXT (ISO 8601) | 消息时间戳 |

### 7.6.2 写入时序

每次 `/api/chat` 请求产生以下写操作：

```
1. store_chat_session(session_id, title, latest_question)
   └── INSERT ... ON CONFLICT(id) DO UPDATE SET updated_at, last_question
       # 首次创建会话，后续只更新时间和最后问题

2. append_chat_message(session_id, "user", latest_question)
   └── INSERT INTO chat_messages (session_id, role, content, sources_json, follow_ups_json, created_at)
       # sources_json = "[]", follow_ups_json = "[]"

3. append_chat_message(session_id, "assistant", answer, sources, follow_up_questions)
   └── INSERT INTO chat_messages (...)
       # sources_json = JSON 序列化的引用文章列表
       # follow_ups_json = JSON 序列化的建议问题列表
   └── UPDATE chat_sessions SET updated_at = ? WHERE id = ?
       # 每条消息写入后刷新会话时间戳
```

### 7.6.3 读取接口

**列表查询 `list_chat_sessions()`：**
- 按 `updated_at DESC` 排序。
- 限制最多返回 **30** 条。
- 返回字段：`session_id`、`title`、`updated_at`、`last_question`。

**详情查询 `get_chat_session_detail(session_id)`：**
- 查询会话元信息 + 所有消息（按 `id ASC` 排序）。
- 每条消息的 `sources_json` 和 `follow_ups_json` 实时反序列化。
- 会话不存在时返回 HTTP 404。

**删除 `delete_chat_session(session_id)`：**
- 先删 `chat_messages` 中该 session 的所有记录，再删 `chat_sessions` 本行。
- 会话不存在时返回 HTTP 404。
- 返回 `{"session_id": "...", "deleted": true}`。

## 7.7 后续问题建议生成

### 7.7.1 自由问答的 `_build_follow_ups`

```python
def _build_follow_ups(question, sources, language):
    topic = _compact_follow_up_subject(
        _first_topic(sources, question),   # 优先取源文章的 main_topic
        question,                          # 回退到用户问题
        language,
    )
    if not topic:
        return ["/brief AI战略", "/今日一读", "/继续阅读"]  # 默认建议（中文示例）
    return _build_command_follow_ups(topic, language)
```

### 7.7.2 `_compact_follow_up_subject` 主题压缩

将主题文本压缩到适合命令参数的长度：
- **中文**：在第一个句号/逗号/分号等标点处截断，上限 **16 字符**。
- **英文**：在第一个 `.!?;:` 处截断，上限 **44 字符**，并在最近的空格处断词。

### 7.7.3 `_build_command_follow_ups` 命令建议生成

根据压缩后的主题，固定生成三条建议：

```python
[
    "/brief {subject}",     # 或 "/简报 {subject}"
    "/timeline {subject}",  # 或 "/时间线 {subject}"
    "/recommend {subject}", # 或 "/继续阅读 {subject}"
]
```

命令标签根据 `language` 自动在中英文之间切换。

### 7.7.4 命令流程的 `follow_up_questions`

各命令在返回时硬编码自己的后续建议，通常为三条互补命令。具体见 7.4 节各命令的"后续建议"说明。

## 7.8 置信度计算规则总表

| 场景 | 置信度 | 计算方式 |
|------|--------|---------|
| `/brief` --- 有源文章 | **0.88** | 固定值 |
| `/brief` --- 无源文章 | 0.30 | 固定值 |
| `/summarize` --- 有源文章 | **0.85** | 固定值 |
| `/summarize` --- 无源文章 | 0.30 | 固定值 |
| `/timeline` --- 有源文章 | **0.83** | 固定值 |
| `/timeline` --- 无源文章 | 0.30 | 固定值 |
| `/compare` --- 有源文章 | **0.84** | 固定值 |
| `/compare` --- 无源文章 | 0.30 | 固定值 |
| `/today` --- 有源文章 | **0.88** | 固定值 |
| `/today` --- 无源文章 | 0.45 | 固定值 |
| `/recommend` --- 有源文章 | **0.80** | 固定值 |
| `/recommend` --- 无源文章 | 0.35 | 固定值 |
| 命令格式不完整 | **0.20** | 固定值 |
| 不支持的命令 | **0.20** | 固定值 |
| 自由问答 --- 有源文章 | **动态** | `min(1.0, sources[0]["score"] / 12)` |
| 自由问答 --- 无源文章 | 0.45 | 固定值 |

**自由问答动态置信度说明：** 取最相关文章的原始检索分数除以 12，再钳制到 `[0, 1.0]`。例如检索分数 9.6 对应置信度 0.8，分数 12+ 对应 1.0。

## 7.9 降级与容错

### 7.9.1 `_fallback_answer` 降级回答

当 `ai_service.is_ai_enabled()` 为 `False` 或 `answer_with_sources` 调用抛出异常时，`_answer_from_sources` 退回到 `_fallback_answer`：

- **无源文章**：返回"当前知识库里暂时没有找到足够相关的业务材料。建议换一个更具体的主题继续问..."
- **有源文章**：返回"基于当前检索到的材料，可以先从下列阅读路径入手"，列出最多 4 篇文章的标题、日期和摘要，末尾提示可继续做简报/时间线/对比。

### 7.9.2 命令参数缺失的 `_command_usage`

返回格式化的用法提示，附带三条默认建议命令，置信度固定 0.2。

## 7.10 上下文构建细节

### 7.10.1 `_build_context_blocks` --- 检索材料格式化

将检索结果列表转换为 LLM 可读的文本块，传入 `answer_with_sources` 的 `context_blocks` 参数：

```
[1] Title: 生成式AI在金融行业的落地路径
Date: 2025-03-15
Topic: AI与金融科技
Excerpt: 复旦管院最新报告指出...

[2] Title: OpenAI与谷歌的大模型竞争格局
Date: 2025-03-10
Topic: 大模型竞争
Excerpt: 2025年Q1以来...
```

### 7.10.2 `history_text` --- 对话历史

从 `request.messages` 的最后 **6 条**消息中按 `"{role}: {content}"` 格式拼接，用换行分隔。超出 6 条的早期消息被截断。

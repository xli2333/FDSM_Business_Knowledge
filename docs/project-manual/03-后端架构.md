## 1. FastAPI 应用初始化

应用入口位于 `backend/main.py`。初始化过程分为 **模块级预处理** 和 **app 对象创建** 两个阶段。

### 1.1 模块级预处理

```python
from backend.config import ALLOWED_ORIGINS, APP_TITLE, MEDIA_UPLOADS_DIR
from backend.database import ensure_database_ready, ensure_runtime_tables
from backend.services.media_service import sync_local_audio_library

ensure_database_ready()          # 步骤 1：数据库就绪检查
ensure_runtime_tables()          # 步骤 2：运行时表创建与迁移
MEDIA_UPLOADS_DIR.mkdir(parents=True, exist_ok=True)  # 确保上传目录存在
sync_local_audio_library()       # 步骤 3：本地音频同步
```

以上代码在 **模块导入阶段** 执行，即 Uvicorn 加载 `backend.main` 时立即运行，早于 `app` 对象创建。

### 1.2 app 对象创建

```python
app = FastAPI(title=APP_TITLE)
```

|------|-----|------|

1. 添加 CORS 中间件

4. 定义根路径健康检查端点 `GET /`

## 2. 启动三步骤

应用启动时按严格顺序执行三个初始化步骤。任何步骤失败将阻止 app 对象创建，服务不会接受请求。

### 2.1 步骤 1 — `ensure_database_ready()`

**文件**: `backend/database.py`

**职责**: 检查 SQLite 数据库文件是否存在，且包含 11 张核心表。如果缺失或不完整，自动触发全量重建。

#### 11 张核心表

| # | 表名 | 职责 |
|---|------|------|
| 1 | `articles` | 文章主表 |
| 2 | `tags` | 标签表 |
| 3 | `article_tags` | 文章-标签关联表 |
| 4 | `columns` | 栏目表 |
| 5 | `article_columns` | 文章-栏目关联表 |
| 6 | `topics` | 专题表 |
| 7 | `topic_articles` | 专题-文章关联表 |
| 8 | `topic_tags` | 专题-标签关联表 |
| 9 | `featured_articles` | 首页推荐文章 |
| 10 | `chat_sessions` | AI 对话会话 |
| 11 | `chat_messages` | AI 对话消息 |

#### 检查逻辑

```
REQUIRED_TABLES = { 上述 11 张表 }

def database_is_ready() -> bool:
    1. 检查 SQLITE_DB_PATH 文件是否存在
    2. 查询 sqlite_master 获取所有 table 和 view
    3. 返回 REQUIRED_TABLES.issubset(existing)

def ensure_database_ready(force_rebuild=False):
    如果 force_rebuild 或 not database_is_ready():
        调用 backend.scripts.build_business_db.rebuild_database()
```

#### 数据库连接配置

```python
def get_connection() -> sqlite3.Connection:
    connection = sqlite3.connect(SQLITE_DB_PATH, timeout=60)
    connection.row_factory = sqlite3.Row      # 行以字典方式访问
    connection.execute("PRAGMA busy_timeout = 60000")  # 等待锁 60 秒
    return connection
```

- **连接超时**: 60 秒
- **busy_timeout**: 60000 毫秒
- **row_factory**: `sqlite3.Row`（字典访问模式）
- **连接管理**: 提供 `connection_scope()` 上下文管理器，自动关闭连接

使用 `_table_has_column()` 辅助函数判断列是否存在后，条件性执行 `ALTER TABLE ADD COLUMN`。这保证了对已有数据库的 **向前兼容** 迁移，无需手动 schema 管理工具。

| `chat_messages` | `follow_ups_json` |

#### 2.2.3 数据修复与补齐

迁移之后，还执行一系列 `UPDATE` 语句进行数据修复：

- `articles.access_level` 空值回填为 `'public'`
- `editorial_articles.access_level` 空值回填为 `'public'`
- `editorial_articles.source_markdown` 空值回填为 `content_markdown`
- `editorial_articles.layout_mode` 空值回填为 `'auto'`
- `editorial_articles.workflow_status` 与 `status` 同步（已发布文章同步为 `'published'`）
- `media_items` 回填 `series_name`、`episode_number`、`preview_url`、`transcript_markdown`、`chapter_payload_json`

#### 2.2.4 种子数据

- **billing_plans**: 从 `config.py` 的 `BILLING_PLAN_DEFINITIONS` 插入 3 个计费套餐（free_member_monthly / paid_member_monthly / paid_member_yearly），使用 `ON CONFLICT DO UPDATE` 实现幂等
- **media_items**: 当表为空时，播种 6 个媒体样本（3 个音频 + 3 个视频），分别覆盖 public / member / paid 三种可见性
- **business_users**: 调用 `ensure_business_user_seed_state()` 播种种子用户

## 3. CORS 配置

**文件**: `backend/main.py`

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=ALLOWED_ORIGINS or ["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 配置项详解

| 参数 | 值 | 说明 |
|------|-----|------|
| `allow_origins` | `ALLOWED_ORIGINS` 或 `["*"]` | 从环境变量 `ALLOWED_ORIGINS` 读取，逗号分隔；未设置时允许所有来源 |
| `allow_credentials` | `True` | 允许发送 Cookie 和 Authorization 头 |
| `allow_methods` | `["*"]` | 允许所有 HTTP 方法 |
| `allow_headers` | `["*"]` | 允许所有请求头 |

### 环境变量

```bash
# .env 文件
ALLOWED_ORIGINS=http://localhost:5173,http://localhost:4173,https://fdsm.example.com
```

`ALLOWED_ORIGINS` 在 `backend/config.py` 中解析：

```python
ALLOWED_ORIGINS = [
    origin.strip()
    for origin in os.getenv("ALLOWED_ORIGINS", "*").split(",")
    if origin.strip()
]
```

**注意**: 当 `allow_credentials=True` 时，浏览器不接受 `allow_origins=["*"]`，此时应配置具体的域名列表。代码中使用 `ALLOWED_ORIGINS or ["*"]` 确保环境变量为空时不会产生空列表。

## 4. 静态文件挂载

**文件**: `backend/main.py`

```python
audio_directory = Path(__file__).resolve().parent.parent / "audio"

app.mount("/audio-files", StaticFiles(directory=audio_directory, check_dir=False), name="audio-files")
app.mount("/media-uploads", StaticFiles(directory=MEDIA_UPLOADS_DIR, check_dir=False), name="media-uploads")
```

| URL 前缀 | 文件系统路径 | 用途 | `check_dir` |
|----------|-------------|------|-------------|
| `/audio-files` | `<项目根>/audio/` | 本地音频文件直接访问 | `False` |
| `/media-uploads` | `<DATA_DIR>/uploads/media/` | 用户/管理员上传的媒体文件 | `False` |

**`check_dir=False`** 表示即使目录不存在也不抛出启动异常，这对于首次部署时目录尚未创建的情况是必要的。

`MEDIA_UPLOADS_DIR` 的定义路径：

```python
# backend/config.py
DATA_DIR = Path(os.getenv("FDSM_DATA_DIR", BASE_DIR))
UPLOADS_DIR = DATA_DIR / "uploads"
MEDIA_UPLOADS_DIR = UPLOADS_DIR / "media"
```

## 6. 请求认证流程

### 6.1 认证核心函数

认证逻辑集中在 `backend/services/supabase_auth_service.py` 的 `get_authenticated_user()` 函数中。

### 6.2 完整流程图

```

    │
    ▼

│  提取 Authorization Header       │

                                        │
                                   id 存在?
                                   ├── 否 → return None
                                   └── 是 → return 用户对象
```

### 6.3 DEV_AUTH 开发模式

当 `DEV_AUTH_ENABLED=1` 且请求携带 `X-Debug-User-Id` 头时，跳过 Supabase 验证，直接构造调试用户：

```python
def _build_debug_user(debug_user_id, debug_user_email):
    if not DEV_AUTH_ENABLED or not debug_user_id:
        return None
    return {
        "id": debug_user_id.strip(),
        "email": debug_user_email.strip() if debug_user_email else None,
        "raw_user": {"debug": True},
    }
```

触发条件：

1. Supabase 未配置 → 直接走 DEV_AUTH
2. Supabase 已配置但无 Bearer token → 降级走 DEV_AUTH
3. DEV_AUTH_ENABLED=0 → 返回 None（匿名用户）

### 6.4 认证状态聚合

`get_auth_status_payload()` 是认证的高阶封装，在 `/api/auth/status` 中调用，返回完整的认证态：

## 7. 错误处理

### 7.1 HTTPException 使用规范

项目全局使用 FastAPI 的 `HTTPException` 进行错误返回，按状态码分类：

| 状态码 | 用途 | 典型场景 |
|--------|------|---------|
| **400** Bad Request | 参数校验失败 | 邮箱格式错误（`commerce`）、翻译参数异常（`articles`）、无效的媒体类型 |
| **401** Unauthorized | 未认证 | 需要登录的操作（点赞、收藏、个人库）、密码登录失败 |
| **403** Forbidden | 权限不足 | 非管理员访问管理接口（`require_admin_profile`）、非付费会员访问智能搜索（`require_paid_profile`） |
| **404** Not Found | 资源不存在 | 文章/媒体/机构/专题不存在 |
| **503** Service Unavailable | 外部服务不可用 | AI 翻译服务调用失败 |

### 7.2 AI 服务容错机制

**文件**: `backend/services/ai_service.py`

AI 服务基于 Google Gemini API，实现了三层容错：

#### 7.2.1 多 Key 轮转

```python
GEMINI_API_KEYS = [item.strip() for item in os.getenv("GEMINI_API_KEYS", "").split(",")]
PRIMARY_GEMINI_KEY = GOOGLE_API_KEY or (GEMINI_API_KEYS[0] if GEMINI_API_KEYS else "")
```

- 支持配置多个 API Key，通过 `GEMINI_API_KEYS` 环境变量传入（逗号分隔）
- `get_gemini_api_keys()` 合并去重所有 Key，返回不可变元组
- 请求时使用全局计数器 `_gemini_request_counter` 轮转起始 Key
- 每次重试自动切换到下一个 Key：`api_key = api_keys[(start_offset + attempt) % len(api_keys)]`
- 最大重试次数：`max(3, len(api_keys) * 2)`

#### 7.2.2 指数退避重试

```python
_GEMINI_RETRYABLE_STATUS = {429, 500, 502, 503, 504}
_GEMINI_TIMEOUT = (10, 90)  # (connect_timeout, read_timeout)

for attempt in range(max_attempts):
    try:
        response = session.post(url, json=payload, timeout=_GEMINI_TIMEOUT)
        if response.status_code == 200:
            return result
        if response.status_code in _GEMINI_RETRYABLE_STATUS:
            time.sleep(min(4.0, 0.6 * (attempt + 1)))  # 线性退避，上限 4 秒
            continue
        raise RuntimeError(...)  # 不可重试的错误直接抛出
    except requests.RequestException:
        time.sleep(min(4.0, 0.6 * (attempt + 1)))
        continue
```

**可重试状态码**: 429 (Rate Limit)、500、502、503、504

**退避策略**: `min(4.0, 0.6 * (attempt + 1))`，即 0.6s → 1.2s → 1.8s → 2.4s → ... → 4.0s（上限）

#### 7.2.3 优雅降级

```python
# 1. AI 未配置时的检查
def is_ai_enabled() -> bool:
    return bool(get_gemini_api_keys())

# 2. 搜索查询扩展降级
def expand_query(query):
    if not is_ai_enabled():
        return []           # AI 不可用时返回空列表，不影响基础搜索
    try:
        ...
    except Exception:
        return []           # 任何异常都不中断搜索

# 3. 摘要降级
# 如果 AI 预生成摘要可用且 hash 匹配，直接返回缓存
# 否则实时调用 AI 生成（可能失败时向上层抛 503）
```

### 7.3 数据库错误处理

- **连接超时**: `sqlite3.connect(timeout=60)` + `PRAGMA busy_timeout = 60000`，等待写锁最长 60 秒
- **连接管理**: 所有数据库操作通过 `connection_scope()` 上下文管理器，确保连接关闭
- **幂等迁移**: `CREATE TABLE IF NOT EXISTS` + `_table_has_column()` 检查，重复执行不报错
- **事务**: `ensure_runtime_tables()` 在最后统一调用 `connection.commit()`

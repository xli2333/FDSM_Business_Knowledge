# 01 · 前置准备与环境校验

> **前置依赖**：无（起点）
> **预计耗时**：2-4 小时（含 Docker Desktop 下载）
> **完成标志**：
> - `docker version` 显示 `OS/Arch: linux/amd64`
> - `data/fudan_knowledge_base.db` 存在且大小 ~704MB
> - Git 分支 `deployment/docker-prod` 已创建并 checkout
> - `requirements.lock.txt` 已生成且无 Windows 专属包

---

## 背景与目标

这一步把开发机调整到能跑 Docker 生产级栈的状态。关键有四件事：

1. **Docker Desktop 装好并切到 Linux 容器模式**（Windows 默认也可能是 Windows 容器模式，要显式切）
2. **项目目录重组**：原来散在项目根的 `fudan_knowledge_base.db`、`faiss_index_business/`、`uploads/`、`audio/` 移进 `data/` 子目录，为 volume 挂载做准备
3. **生成 `requirements.lock.txt`**：当前 `requirements.txt` 只有 11 行（严重不全），必须从本地 venv 导出完整依赖
4. **Git 分支管理**：所有改动在新分支 `deployment/docker-prod` 上做，随时能回滚

做完这步你的工作目录结构会和最终生产部署的目录结构**完全一致**，之后每步改造都是纯代码和纯配置变更。

## 原子步骤

### 步骤 1 · 校验 Docker Desktop

**操作**：
```powershell
# 在 PowerShell 里跑
docker --version
docker compose version
docker info --format "{{.OSType}}"
```

**验证**：
- `docker --version` 应返回 `>= 24.0.x`
- `docker compose version` 应返回 `>= v2.20`
- `docker info --format "{{.OSType}}"` 必须返回 **`linux`**（不是 `windows`）

**如果返回 `windows`**：
1. 右键任务栏 Docker Desktop 图标 → **Switch to Linux containers**
2. 等 30 秒切换完成，重新跑 `docker info --format "{{.OSType}}"` 确认

**如果 Docker Desktop 没装或版本太低**：
- 下载：https://www.docker.com/products/docker-desktop
- 安装时必须勾选 **"Use WSL 2 instead of Hyper-V"**
- 装完后在 Settings → General 里确认 **"Use the WSL 2 based engine"** 打勾

**常见错误**：
| 错误 | 原因 | 修复 |
|---|---|---|
| `docker: command not found` | Docker Desktop 没启动 | 启动 Docker Desktop，等任务栏图标变绿 |
| `error during connect: ...pipe/dockerDesktopLinuxEngine` | WSL2 没启动 | `wsl --install` 然后重启 |
| 显示 Windows 容器 | 错了模式 | 右键图标切 Linux |

### 步骤 5 · 移动数据到 `data/`

⚠️ **这一步改变了文件位置**。`backend/config.py` 已经支持 `FDSM_DATA_DIR` 环境变量切换数据目录，所以只要 env 设对，代码不用改。

**操作前先确认当前数据文件位置**：
```powershell
Get-Item fudan_knowledge_base.db | Select-Object Name, Length
# 应显示 ~704 MB

Get-ChildItem -Directory faiss_index_business | Measure-Object -Sum Length
```

**移动**：
```powershell
# 移主 DB（以及可能存在的备份 DB）
Move-Item -Force fudan_knowledge_base.db data\
Get-ChildItem *.db -ErrorAction SilentlyContinue | Move-Item -Destination data\ -Force

# 移向量索引
Move-Item -Force faiss_index_business data\

# 如果存在这些目录，也移走（有就移，没有就跳）
if (Test-Path uploads)          { Move-Item -Force uploads data\ }
if (Test-Path audio)            { Move-Item -Force audio data\ }
if (Test-Path rag_chunk_index)  { Move-Item -Force rag_chunk_index data\ }
```

**验证**：
```powershell
Get-ChildItem data\
# 应看到：
# fudan_knowledge_base.db
# faiss_index_business/
# uploads/ （可能有）
# audio/ （可能有）

# 确认根目录已经没有这些了
Get-Item fudan_knowledge_base.db -ErrorAction SilentlyContinue
# 应无输出
```

**⚠️ 临时运行项目不会报错吗？**

会。因为 `backend/config.py` 里 `DATA_DIR = Path(os.getenv("FDSM_DATA_DIR", BASE_DIR))` 默认是项目根。本地直接跑 Python 会找不到 DB。

**处理方式**：
- 本地直接跑 Python 时，先在 PowerShell 设置：
  ```powershell
  $env:FDSM_DATA_DIR="C:\Users\LXG\fdsmarticles\data"
  python -m uvicorn backend.main:app
  ```
- 或者在项目根建一个 `.env.local` 文件（仅本地 Python 用），内容：
  ```
  FDSM_DATA_DIR=C:/Users/LXG/fdsmarticles/data
  ```
  （`python-dotenv` 的 `load_dotenv()` 在 `config.py` 里已经会读它）

但**之后所有运行都在 Docker 里**，容器内 `FDSM_DATA_DIR=/data`，宿主机 `./data/` 挂过去。本地 Python 直接运行会慢慢被淘汰。

### 步骤 6 · 生成 `requirements.lock.txt`

**背景**：当前 `requirements.txt` 只有 11 行，实际依赖远多于此（Grep 一下 `import` 语句就知道还有 `beautifulsoup4`、`lxml`、`numpy`、`pillow`、`markdown` 等）。Docker 构建必须用锁版本，保证每次构建一致。

**操作**：
```powershell
cd C:\Users\LXG\fdsmarticles

# 激活项目 venv（如果是 anaconda 或 poetry 按你的实际方式激活）
.\.venv\Scripts\Activate.ps1

# 导出完整依赖
pip freeze > requirements.lock.raw.txt

# 过滤 Windows 专属包（Linux 容器不能用）
Get-Content requirements.lock.raw.txt |
    Where-Object {
        $_ -notmatch "^pywin32" -and
        $_ -notmatch "^pypiwin32" -and
        $_ -notmatch "^pywinpty" -and
        $_ -notmatch "^-e\s" -and
        $_ -notmatch "^#"
    } |
    Set-Content requirements.lock.txt

# 补加生产必需包（当前 venv 里可能还没装，先加占位）
# 之后 Dockerfile 构建时 pip install -r 会一起装
Add-Content requirements.lock.txt @"
gunicorn==22.0.0
PyJWT==2.9.0
redis==5.2.0
"@

Remove-Item requirements.lock.raw.txt
deactivate
```

**验证**：
```powershell
# 文件应该有 30-80 行
(Get-Content requirements.lock.txt).Count

# 确认关键包都在
Get-Content requirements.lock.txt | Select-String "fastapi|uvicorn|langchain|faiss-cpu|gunicorn|redis"

# 确认 Windows 包都没了
Get-Content requirements.lock.txt | Select-String "pywin"
# 应无输出
```

**常见错误**：
| 症状 | 原因 | 修复 |
|---|---|---|
| venv 激活失败 | Script 执行策略 | `Set-ExecutionPolicy -Scope Process RemoteSigned`，再激活 |
| `pip freeze` 几乎为空 | venv 是新的 | 先 `pip install -r requirements.txt` 装一遍，再 freeze |
| 输出里有 `-e git+...` | 本地开发包（editable install）| 手动删掉那几行 |

**关键包清单参考**（lock 文件里必须有这些，版本可能略有差异）：
```
fastapi>=0.115
uvicorn>=0.32
gunicorn==22.0.0
pydantic>=2.10
sqlalchemy（如果用到）
requests>=2.32
python-dotenv>=1.0
python-multipart>=0.0.20
PyJWT==2.9.0
redis==5.2.0

# AI / RAG
langchain>=0.3
langchain-google-genai>=2.0
langchain-community>=0.3
google-genai>=0.3
faiss-cpu>=1.9
numpy>=1.26

# 内容处理
beautifulsoup4>=4.12
lxml>=5.0
markdown>=3.7
pillow>=11.0
```

Docker 镜像层 + WAL 文件 + 日志 会占用 5-10 GB，先确认项目盘有足够空间：

## 常见错误与排查

| 症状 | 原因 | 修复 |
|---|---|---|
| `Move-Item` 说文件被占用 | 有 Python 进程在用 DB | 关掉正在运行的 `uvicorn` 或 `python` |
| `pip freeze` 导出了一堆和项目无关的包 | 用的不是项目 venv | 确认 venv 路径：`(Get-Command python).Path` |
| Git `checkout` 提示有未提交改动 | 本来有修改 | `git stash` 暂存，或先 `git add + commit` |
| `docker run hello-world` 报错 | Docker 没启动或 WSL2 没起 | 启动 Docker Desktop，等绿灯 |
| `docker info` 显示 OSType = windows | 容器模式错了 | 右键图标切换 Linux 容器 |
| 运行 Python 报 `FileNotFoundError: fudan_knowledge_base.db` | 数据已移动但 env 没设 | `$env:FDSM_DATA_DIR="C:/Users/LXG/fdsmarticles/data"` |
| 硬盘空间不足 | | `docker system prune -af` + 清 C 盘临时文件 |

## 已对齐的当前实现

- 后端镜像文件是 `Dockerfile.backend`。
- 前端镜像文件是 `frontend/Dockerfile`，不是根目录 `Dockerfile.frontend`。
- 本地 Docker 验收使用 `docker-compose.yml` + `.env.docker`。
- 私有云生产部署使用 `docker-compose.prod.yml` + `.env.production`。
- 生产对外端口由宿主机 Nginx 终止 HTTPS，反代到 `127.0.0.1:${APP_PORT}`；容器内前端 Nginx 只监听 HTTP 80。
- 生产 compose 默认 `127.0.0.1:${APP_PORT:-8080}:80`，不直接把容器 80 暴露到公网。
- 静态资源在前端 Nginx 容器中直出：`/audio-files/`、`/editorial-uploads/`、`/media-uploads/`。
- CAS 生产变量以 `.env.production.example` 为准：`AUTH_BACKEND=cas`、`CAS_ENABLED=1`、`CAS_SERVICE_URL=https://域名/api/auth/cas/callback`。
- 前端生产构建不再需要 `VITE_SUPABASE_URL` 或 `VITE_SUPABASE_ANON_KEY`；CAS 登录由后端接口和同域 `/api` 完成。
- 后端生产默认禁止 Supabase/dual 历史鉴权路径；只有 `ALLOW_LEGACY_SUPABASE_AUTH=1` 才能作为受控回滚启用。
- 付费音视频播放使用 `/api/media/{kind}/{slug}/stream?token=...` 兼容浏览器原生播放器；Nginx 与 Gunicorn 日志均不记录 query token。
- 后端只暴露 `/api/metrics`，根路径 `/metrics` 已删除；宿主机 Nginx 与前端容器 Nginx 均对 `/api/metrics` 加 IP allowlist。
- blue-green 部署默认启用 schema guard；涉及 SQLite schema 的版本必须走维护窗口非并行部署，或人工设置 `ALLOW_SCHEMA_BLUE_GREEN=1`。
- 本地 Docker 默认 CAS 关闭，以 `.env.docker.example` 为准：`AUTH_BACKEND=auto`、`CAS_ENABLED=0`。
- 高并发 AI 操作走 Redis + `backend-worker`，前端编辑台轮询异步任务状态。
- 首页公开 feed 默认启用 `HOME_FEED_CACHE_TTL_SECONDS=30` 短缓存。
- SQLite 已启用 WAL、busy timeout、写事务重试和写入探针。
- 最终压测脚本是 `deploy/load_test.py`；数据库写入压测是 `backend.scripts.concurrency_write_probe`。
- 发布包由 `deploy/create_release_package.ps1` 生成，只包含代码、示例 env、部署脚本和文档，不包含真实 `.env`、数据库、上传资产、node_modules、构建产物。

## 旧文档中不再作为执行依据的内容

以下内容在 `docs/deployment_plan/00-12` 和 `docs/云端部署方案_Docker上云完整指南.md` 中仍作为早期设计稿出现。它们可以保留作设计脉络，但不应按原样执行：

- `Dockerfile.frontend`：已改为 `frontend/Dockerfile`。
- `docker-compose.override.yml`：当前没有使用本地 override 作为部署主线。
- `.env.example`、`.env.development`、直接提交 `.env.production`：当前使用 `.env.docker.example` 与 `.env.production.example`，真实 `.env.*` 不入发布包。
- `deploy/nginx.conf`、`deploy/default.conf`、`deploy/_proxy_base.conf`：当前容器内前端 Nginx 配置为 `deploy/nginx/default.conf`，宿主机 HTTPS 配置为 `deploy/nginx/fdsmarticles-https.conf`。
- 容器内直接监听 HTTPS 443、`HTTPS_PORT`、证书 volume：当前改为宿主机 Nginx 终止 HTTPS。
- `IMAGE_TAG`、`docker compose pull` 镜像仓库发布流程：当前发布包路线是在服务器上 `docker compose --env-file .env.production -f docker-compose.prod.yml build`，后续如果接镜像仓库再单独增加镜像发布文档。
- 早期“CAS 上线后再迁”的表述：当前 CAS 代码已接入，生产默认可以开启；真实上线还取决于学校 CAS 白名单、正式域名和管理员工号配置。
- 早期“debug header 风险待修”的表述：当前生产环境已在 FastAPI middleware 拒绝 `X-Debug-User-Id` 和 `X-Debug-User-Email`。
- 早期“AI 同步阻塞”的表述：编辑台 AI 自动摘要、格式化、翻译已有 Redis 异步任务路径。
- 早期“前端构建依赖 Supabase URL/anon key”的表述：当前前端 Supabase SDK 已退场，不再注入 `VITE_SUPABASE_*`。
- 早期“生产可以直接切 Supabase/dual”的表述：当前必须显式设置 `ALLOW_LEGACY_SUPABASE_AUTH=1`，否则启动和 preflight 都会阻止。
- 早期“音视频 stream 只检查 Authorization header”的表述：当前 stream 路由已支持受控 query token，专供原生 `<audio>`/`<video>` 播放。
- 早期“`/metrics` 与 `/api/metrics` 并存”的表述：当前只保留 `/api/metrics`。

## 上云时容易漏改的点

- 如果 `.env.production` 的 `APP_PORT` 不是 `8080`，必须同步修改 `deploy/nginx/fdsmarticles-https.conf` 中的 `server 127.0.0.1:8080;`。
- `SITE_BASE_URL`、`ALLOWED_ORIGINS`、`CAS_SERVICE_BASE_URL`、`CAS_SERVICE_URL` 必须使用同一个正式 HTTPS 域名。
- CAS 登录带 `redirect` 参数时，后端校验使用的 service URL 会包含 `redirect` 查询参数；学校 CAS 白名单需要允许该回调 URL 模式。
- `data/` 必须手工上传，不在发布包内；至少包含 `fudan_knowledge_base.db`、`Fudan_Business_Knowledge_Data/`、`faiss_index_business/`、`uploads/`、`audio/`。
- Windows 本机默认 `bash.exe` 可能指向不可用 WSL stub；本地验证脚本语法可使用 Git Bash，私有云 Linux 主机必须直接执行 `deploy/acceptance_check.sh`。
- 任何包含 `backend/database.py`、`ensure_runtime_tables`、DDL 或 schema/migration 路径变更的版本，不要直接跑默认 blue-green；先设置 `SCHEMA_CHANGE_REQUIRED=1` 让脚本拒绝并行发布，再按维护窗口流程停旧起新。

# 24 备份恢复演练

本文承接 round186，用于验证 SQLite 备份不只是“创建成功”，而是可校验、可恢复、可回滚。

## 新增脚本

`deploy/restore_sqlite_backup.py` 支持：

- `.db`
- `.db.gz`
- `.db.gpg`
- `.db.gz.gpg`

默认只做校验，不替换数据库：

```bash
python deploy/restore_sqlite_backup.py backups/fudan_knowledge_base.20260421-030000.db.gz
```

真实恢复必须显式加 `--restore`：

```bash
python deploy/restore_sqlite_backup.py backups/fudan_knowledge_base.20260421-030000.db.gz \
  --target-db data/fudan_knowledge_base.db \
  --restore
```

恢复前如果目标库已存在，脚本会生成：

```text
data/fudan_knowledge_base.db.pre-restore-YYYYMMDD-HHMMSS
```

用于快速回退。

## 加密备份

如果备份以 `.gpg` 结尾，脚本会调用：

```bash
gpg --batch --yes --decrypt --output <temp-file> <backup-file>
```

也可指定命令：

```bash
python deploy/restore_sqlite_backup.py backups/fudan_knowledge_base.db.gz.gpg --gpg-command gpg
```

## 云端验收要求

上线前至少执行一次：

1. 创建一份真实备份。
2. 校验备份 quick_check。
3. 恢复到临时路径。
4. 对临时库运行 `PRAGMA quick_check`。
5. 记录恢复耗时、备份文件名、恢复前副本路径。

生产库真实替换只允许在维护窗口或故障恢复窗口执行。

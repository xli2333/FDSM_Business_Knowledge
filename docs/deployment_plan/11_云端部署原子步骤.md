## 背景

本地 17 项全绿说明**镜像和代码是干净的**。云端部署只做四件事：

1. **搭基础设施**：一台 Ubuntu 22.04 + Docker
2. **传代码和镜像**：Git + 镜像仓库
3. **传数据**：SQLite + FAISS + uploads
4. **填 `.env.production` 里的生产值 + 签 SSL**：改完 `docker compose up -d`

**不涉及任何代码改动**。如果发现要改代码，立刻停止——回本地改完过 §10 再来。

## 阶段 A · 服务器就绪

### A1 · 服务器选型与购买

**推荐配置**（复核 §02 章节）：
- 8 核 / 16 GB 内存 / 200 GB SSD
- 10 Mbps 带宽
- Ubuntu 22.04 LTS
- **区域**：Singapore 或 HongKong（Gemini 国内不通）

**阿里云示例**：
- 实例类型：`ecs.c7.2xlarge`
- 镜像：`Ubuntu 22.04 64位`
- 存储：ESSD PL1 200 GB
- 带宽：按固定 10 Mbps 包月

记下三件事：
- 公网 IP（例如 `1.2.3.4`）
- root 密码 或 SSH key
- 实例 ID

### A2 · DNS 解析

在你管理 `fdsm.fudan.edu.cn` DNS 的地方（或者学院 IT 批准你管理子域）：

- 类型：A
- 名称：`knowledge`
- 值：`1.2.3.4`（服务器公网 IP）
- TTL：600（10 分钟，方便后续调整）

**验证生效**（大概 10 分钟后）：
```powershell
nslookup knowledge.fdsm.fudan.edu.cn
# 应返回 1.2.3.4
```

国内 DNS 可能更慢（半小时到几小时）。继续其他步骤不用等。

### A3 · SSH 初连与硬化

**本地生成 SSH key**（如果还没有）：
```powershell
ssh-keygen -t ed25519 -C "fdsm-deploy"
# 默认保存到 ~/.ssh/id_ed25519
```

**把公钥传上服务器**（第一次用密码登录）：
```powershell
# 把公钥拷上去
type $HOME\.ssh\id_ed25519.pub | ssh root@1.2.3.4 "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"
```

**测试 SSH key 登录**：
```powershell
ssh root@1.2.3.4
# 应该免密进去
```

**服务器上做硬化**：
```bash
# 关掉 root 密码登录（只允许 key）
sed -i 's/#PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
sed -i 's/#PermitRootLogin yes/PermitRootLogin prohibit-password/' /etc/ssh/sshd_config
systemctl restart ssh

# 改 SSH 端口（可选，增加攻击成本）
# sed -i 's/#Port 22/Port 2222/' /etc/ssh/sshd_config
# systemctl restart ssh
# ⚠️ 改前先在 ufw 里开新端口，不然会锁死
```

### A4 · 装 Docker + 系统调优

全部在服务器上执行：

```bash
# 更新系统
apt update && apt upgrade -y

# 装 Docker（官方脚本）
curl -fsSL https://get.docker.com | sh
systemctl enable --now docker

# 验证
docker --version
docker compose version

# 建非 root 用户
adduser --disabled-password --gecos "" fdsm
usermod -aG docker fdsm
usermod -aG sudo fdsm

# 把 SSH key 也给新用户
mkdir -p /home/fdsm/.ssh
cp /root/.ssh/authorized_keys /home/fdsm/.ssh/
chown -R fdsm:fdsm /home/fdsm/.ssh
chmod 700 /home/fdsm/.ssh
chmod 600 /home/fdsm/.ssh/authorized_keys

# 防火墙
apt install -y ufw
ufw default deny incoming
ufw default allow outgoing
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw --force enable
ufw status

# 系统参数调优
cat >> /etc/sysctl.conf <<'EOF'

# fdsm-knowledge tuning
vm.swappiness = 10
vm.dirty_ratio = 10
vm.dirty_background_ratio = 5
net.core.somaxconn = 65535
net.core.netdev_max_backlog = 16384
net.ipv4.tcp_max_syn_backlog = 8192
net.ipv4.tcp_fin_timeout = 15
net.ipv4.tcp_tw_reuse = 1
fs.file-max = 1000000
EOF
sysctl -p

# 文件描述符上限
cat >> /etc/security/limits.conf <<'EOF'
* soft nofile 65535
* hard nofile 65535
root soft nofile 65535
root hard nofile 65535
EOF

# 安装常用工具
apt install -y git curl wget jq tmux htop iotop ncdu
```

**切到 fdsm 用户**：
```bash
su - fdsm
```

### A5 · 建工作目录

```bash
sudo mkdir -p /srv/fdsm
sudo chown fdsm:fdsm /srv/fdsm
cd /srv/fdsm
```

## 阶段 B · 镜像与代码上传

### B1 · 本地推镜像到阿里云镜像仓库

**阿里云容器镜像服务（免费）**：

1. 控制台 → 容器镜像服务 → 个人实例（免费）
2. 创建命名空间：`fdsm`
3. 创建镜像仓库：
   - `fdsm-backend`（类型：私有）
   - `fdsm-frontend`（类型：私有）
4. 设置仓库密码（右上角头像 → 实例密码）

**本地推送**：

```powershell
cd C:\Users\LXG\fdsmarticles

# 登录
docker login --username=你的阿里云账号 registry.cn-shanghai.aliyuncs.com
# 提示输入密码 → 就是刚才设的仓库密码

# 打 tag
$TAG = "v1.0.0"
$REGISTRY = "registry.cn-shanghai.aliyuncs.com/fdsm"

docker tag fdsm-knowledge-backend:local "$REGISTRY/fdsm-backend:$TAG"
docker tag fdsm-knowledge-frontend:local "$REGISTRY/fdsm-frontend:$TAG"

# 推
docker push "$REGISTRY/fdsm-backend:$TAG"
docker push "$REGISTRY/fdsm-frontend:$TAG"
```

**推送耗时**：
- backend: 1.2 GB，上行 10 Mbps 约 20 分钟
- frontend: 60 MB，约 1 分钟

**如果慢**：阿里云镜像服务有"海外加速器"功能，推完即可。

**记下镜像地址**（之后服务器 pull 用）：
```
registry.cn-shanghai.aliyuncs.com/fdsm/fdsm-backend:v1.0.0
registry.cn-shanghai.aliyuncs.com/fdsm/fdsm-frontend:v1.0.0
```

### B2 · 服务器 clone 项目（拿配置文件）

**服务器上**：
```bash
cd /srv/fdsm

# clone（用 HTTPS + token，或 SSH deploy key）
git clone https://github.com/你的账号/fdsmarticles.git app
cd app
git checkout deployment/docker-prod

# 生产不用 override
rm -f docker-compose.override.yml

# 删掉本地数据目录占位（data/ 在 .gitignore 所以没进库，只是稳一手）
rm -rf data backups

```

### B3 · 服务器 pull 镜像 + 改 compose 指向仓库

```bash
# 登录镜像仓库
docker login --username=你的阿里云账号 registry.cn-shanghai.aliyuncs.com

# 拉镜像
docker pull registry.cn-shanghai.aliyuncs.com/fdsm/fdsm-backend:v1.0.0
docker pull registry.cn-shanghai.aliyuncs.com/fdsm/fdsm-frontend:v1.0.0

# 验证
docker images | grep fdsm
```

**改 `docker-compose.yml` 让它用仓库镜像**：

有两种做法：

**做法 1 · sed 直改**
```bash
sed -i 's|image: fdsm-knowledge-backend:|image: registry.cn-shanghai.aliyuncs.com/fdsm/fdsm-backend:|g' docker-compose.yml
sed -i 's|image: fdsm-knowledge-frontend:|image: registry.cn-shanghai.aliyuncs.com/fdsm/fdsm-frontend:|g' docker-compose.yml
```

**做法 2 · 用 env 变量**

回本地先改 `docker-compose.yml` 支持：
```yaml
services:
  backend-web:
    image: ${BACKEND_IMAGE:-fdsm-knowledge-backend}:${IMAGE_TAG:-local}
  frontend:
    image: ${FRONTEND_IMAGE:-fdsm-knowledge-frontend}:${IMAGE_TAG:-local}
```

然后 `.env.production` 里设：
```
BACKEND_IMAGE=registry.cn-shanghai.aliyuncs.com/fdsm/fdsm-backend
FRONTEND_IMAGE=registry.cn-shanghai.aliyuncs.com/fdsm/fdsm-frontend
IMAGE_TAG=v1.0.0
```

**推荐做法 2**，便于未来版本切换。本次 commit 一下：
```powershell
# 本地
git add docker-compose.yml
git commit -m "deploy(step-11): parametrize image names for registry pull"
git push
# 然后服务器 git pull
```

## 阶段 E · SSL 证书

### E1 · Let's Encrypt 签发

**前提**：
- DNS 已生效（A 记录指向 1.2.3.4）
- 80 端口能访问（安全组和 ufw 都开了）

**用 certbot 独立容器签发**：

```bash
cd /srv/fdsm/app

# 先临时停 frontend（certbot 要占用 80 端口）
docker compose stop frontend

# 跑 certbot standalone 模式
docker run --rm \
    -p 80:80 \
    -v /srv/fdsm/app/deploy/certbot/conf:/etc/letsencrypt \
    -v /srv/fdsm/app/deploy/certbot/www:/var/www/certbot \
    certbot/certbot certonly --standalone \
    -d knowledge.fdsm.fudan.edu.cn \
    --email admin@fdsm.fudan.edu.cn \
    --agree-tos \
    --no-eff-email \
    --non-interactive

# 验证证书生成
sudo ls -la deploy/certbot/conf/live/knowledge.fdsm.fudan.edu.cn/
# 应该看到：
#   cert.pem, chain.pem, fullchain.pem, privkey.pem
```

**⚠️ 如果签发失败**：
- DNS 没生效 → `nslookup knowledge.fdsm.fudan.edu.cn`
- 80 端口被占用 → `sudo netstat -tlnp | grep :80`
- Let's Encrypt 限流 → 单域每周最多 50 次，同一天同域 5 次失败；换时间再试

### E2 · 启用 HTTPS server 块

**修改 `deploy/default.conf`**，把 §09 里注释的 HTTPS server 块取消注释。

或者简单点——创建一个 `deploy/default.ssl.conf`：

```bash
cat > /srv/fdsm/app/deploy/default.ssl.conf <<'EOF'
# HTTPS 主站配置（仅生产启用）

server {
    listen 443 ssl http2;
    server_name knowledge.fdsm.fudan.edu.cn;

    ssl_certificate /etc/letsencrypt/live/knowledge.fdsm.fudan.edu.cn/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/knowledge.fdsm.fudan.edu.cn/privkey.pem;
    ssl_session_timeout 1d;
    ssl_session_cache shared:SSL:50m;
    ssl_session_tickets off;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;

    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    add_header X-Content-Type-Options nosniff always;
    add_header X-Frame-Options SAMEORIGIN always;
    add_header Referrer-Policy strict-origin-when-cross-origin always;

    limit_conn conn_per_ip 20;

    # ========== 和 HTTP server 块的 location 配置相同 ==========
    root /usr/share/nginx/html;
    index index.html;

    location ~* ^/assets/.+\.(js|css|woff2?|ttf|eot|png|jpg|jpeg|gif|svg|webp|ico|mp4|mp3)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
        access_log off;
        try_files $uri =404;
    }

    location = /favicon.ico {
        expires 30d;
        access_log off;
        try_files $uri =404;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }

    location = /nginx-health {
        access_log off;
        return 200 "ok\n";
        add_header Content-Type text/plain;
    }

    location ~ ^/api/auth {
        limit_req zone=api_auth burst=20 nodelay;
        proxy_pass http://backend;
        include /etc/nginx/snippets/_proxy_base.conf;
    }

    location ~ ^/api/(chat|search|editorial/ai|editorial/summarize|editorial/translate|editorial/format) {
        limit_req zone=api_ai burst=10 nodelay;
        proxy_pass http://backend;
        include /etc/nginx/snippets/_proxy_base.conf;
        proxy_read_timeout 180s;
        proxy_send_timeout 180s;
    }

    location /api/ {
        limit_req zone=api_general burst=50 nodelay;
        proxy_pass http://backend;
        include /etc/nginx/snippets/_proxy_base.conf;
    }

    location /audio-files/ {
        alias /data/audio/;
        expires 30d;
        add_header Cache-Control "public";
        access_log off;
    }

    location /editorial-uploads/ {
        alias /data/uploads/editorial/;
        expires 7d;
        access_log off;
    }

    location /media-uploads/ {
        alias /data/uploads/media/;
        expires 7d;
        access_log off;
    }

    location ~ /\.(?!well-known) {
        deny all;
        access_log off;
        return 404;
    }
}
EOF
```

**修改 HTTP server 块做强制跳转**：

改 `deploy/default.conf`，把 HTTP server 块的 `location /` 改成：
```nginx
location / {
    # 生产：重定向到 HTTPS
    return 301 https://$host$request_uri;
}
```

其他 location（`/api/`, `/audio-files/` 等）也可以**保留 HTTP 版本**用于向后兼容，但建议都返回 301 到 HTTPS。

**把 ssl.conf 拷进 frontend 容器**：

最稳的做法是**改 Dockerfile.frontend**，但那需要重新构建推镜像。更快的做法：挂载到容器内 `/etc/nginx/conf.d/`：

修改 `docker-compose.yml` 的 frontend 服务 volumes：
```yaml
frontend:
  volumes:
    - ./data/audio:/data/audio:ro
    - ./data/uploads:/data/uploads:ro
    - ./deploy/certbot/conf:/etc/letsencrypt:ro
    - ./deploy/certbot/www:/var/www/certbot:ro
    - ./deploy/default.conf:/etc/nginx/conf.d/default.conf:ro          # 新增
    - ./deploy/default.ssl.conf:/etc/nginx/conf.d/default.ssl.conf:ro  # 新增
    - ./deploy/_proxy_base.conf:/etc/nginx/snippets/_proxy_base.conf:ro # 新增
```

这样改 `.conf` 文件不用重建镜像，`docker compose restart frontend` 即可生效。

```bash
cat > /srv/fdsm/app/deploy/backup_loop.sh <<'EOF'
#!/usr/bin/env bash
set -euo pipefail

BACKUP_DIR=/backups
DATA_DIR=/data
RETENTION_DAILY=14
RETENTION_WEEKLY=8

mkdir -p "$BACKUP_DIR/daily" "$BACKUP_DIR/weekly"

while true; do
    TIMESTAMP=$(date +%Y%m%d_%H%M%S)
    TODAY=$(date +%u)

    DAILY_DIR="$BACKUP_DIR/daily/$TIMESTAMP"
    mkdir -p "$DAILY_DIR"
    echo "[$(date)] backing up SQLite ..."

    # 用 sqlite3 .backup 做热备，防 WAL 不一致
    python3 -c "
import sqlite3
src = sqlite3.connect('$DATA_DIR/fudan_knowledge_base.db')
dst = sqlite3.connect('$DAILY_DIR/fudan_knowledge_base.db')
src.backup(dst)
src.close()
dst.close()
"

    if [ "$TODAY" = "7" ]; then
        WEEKLY_DIR="$BACKUP_DIR/weekly/$TIMESTAMP"
        mkdir -p "$WEEKLY_DIR"
        echo "[$(date)] weekly snapshot ..."
        tar --zstd -cf "$WEEKLY_DIR/faiss.tar.zst" -C "$DATA_DIR" faiss_index_business || true
        tar --zstd -cf "$WEEKLY_DIR/uploads.tar.zst" -C "$DATA_DIR" uploads audio || true
        cp "$DAILY_DIR/fudan_knowledge_base.db" "$WEEKLY_DIR/" || true
    fi

    find "$BACKUP_DIR/daily" -maxdepth 1 -type d -mtime +$RETENTION_DAILY -exec rm -rf {} + 2>/dev/null || true
    find "$BACKUP_DIR/weekly" -maxdepth 1 -type d -mtime +$((RETENTION_WEEKLY * 7)) -exec rm -rf {} + 2>/dev/null || true

    echo "[$(date)] backup done: $DAILY_DIR"
    sleep 86400
done
EOF

chmod +x /srv/fdsm/app/deploy/backup_loop.sh
docker compose restart backup
```

## 上线后 24 小时观察

**观察点**：

- 无 traceback
- WAL 文件 < 200MB（涨到 GB 级说明 checkpoint 没触发）

**常见上线 24 小时内的问题**：

| 症状 | 排查 | 修 |
|---|---|---|
| 告警邮件满天飞 | uptime robot 检测间隔太短 | 改成 5 分钟 |
| Gemini 调用频繁 429 | 单 key 限额 | 加 key 到 `GEMINI_API_KEYS` |
| 内存缓慢增长 | 可能 Gemini 客户端泄漏 | gunicorn `--max-requests` 会定期重启 worker |
| 首次搜索极慢 | FAISS 冷启动 | 等 30 秒第二次就快了；可考虑启动后自动预热 |
| DB is locked 偶发 | 信号量还是不够 | §07 `DB_WRITE_CONCURRENCY=2` 再试 |

## 常见错误与排查

| 症状 | 原因 | 修 |
|---|---|---|
| `docker pull` 403 | 镜像仓库没登录 | `docker login` 重来 |
| `docker compose up` 有服务起不来 | 某个 env 变量漏填 | `docker compose config` 看渲染结果 |
| `/api/health` 返回 503 | DB 不可达 / FAISS 不存在 | 检查 volume 挂对没、文件权限 |
| Let's Encrypt timeout | DNS 没生效 / 80 端口被挡 | `nslookup`、`sudo lsof -i :80` |
| HTTPS 证书无效 | 证书文件路径错 | 进 frontend 容器 `ls /etc/letsencrypt/live/` |
| 首页加载但 API 全 502 | backend-web 没 healthy | `docker compose ps` 看状态 |
| 上传文件 413 Request Entity Too Large | `client_max_body_size` 太小 | 回 §09 的 nginx.conf |
| 跨域失败 | `ALLOWED_ORIGINS` 没包含生产域名 | 改 `.env.production` 重启 backend-web |

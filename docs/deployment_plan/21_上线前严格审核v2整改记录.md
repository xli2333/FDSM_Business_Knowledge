# 21 上线前严格审核 v2 整改记录

> 本文承接 `20_上线前严格审核报告v2.md`。v2 报告判断合理：上一轮代码级 P0 大多已闭环，但正式商用还缺 HTTPS 自动化、备份真异地加密、可观测、定时清理和 debug auth 纵深防御。本轮把本地可落地项继续收口，真实云资源项继续作为 preflight 和上线验收硬门槛。

## 已落地整改

- `APP_ENV` 白名单：只允许 `production/staging/development/test`，误写 `prod`、空值或其他值会启动失败。
- debug header 纵深防御：`X-Debug-User-Id/Email` 即使仍作为开发路由参数存在，也只允许 development/test 模式构造预览身份；生产和 staging 不会生效。
- Gunicorn：新增 `GUNICORN_PRELOAD`、`GUNICORN_WORKER_CONNECTIONS` 和包含 `X-Forwarded-For`/耗时的 access log format。
- HTTPS：新增 `deploy/bootstrap_https.sh`，用于真实云主机上按 `DOMAIN/EMAIL/APP_PORT` 签发 certbot 证书、写入 HTTPS Nginx 配置、安装续期 reload hook。
- 备份：`deploy/backup_sqlite.py` 从单纯本机 `.db` 复制升级为 SQLite backup + gzip + GPG 加密 + offsite 推送；支持 `file://`、`rclone copy`、`ossutil cp`；缺命令或配置会失败。
- Backup 镜像：新增 `Dockerfile.backup`，备份容器内置 `gnupg` 和 `rclone`。
- Housekeeping：新增 `backend.scripts.housekeeping` 并接入 compose，默认每小时清理过期/撤销 CAS session。
- 可观测：后端新增 request id、结构化 JSON 请求日志、`/metrics` 和 `/api/metrics` Prometheus 文本指标；支持 `METRICS_TOKEN` 保护。
- Sentry：新增 `SENTRY_DSN` 和 `SENTRY_TRACES_SAMPLE_RATE`，生产可通过 `SENTRY_REQUIRED=1` 让 preflight 强制要求异常追踪。
- Compose resources：保留 `cpus/mem_limit`，同时增加 `deploy.resources.limits/reservations`。
- preflight：新增 `Dockerfile.backup`、`bootstrap_https.sh`、`METRICS_TOKEN`、`SENTRY_DSN`、备份 offsite/encryption 占位值校验。

## 云端必须真实完成

- `SITE_BASE_URL`、`ALLOWED_ORIGINS`、`CAS_SERVICE_BASE_URL` 必须替换为正式 HTTPS 域名。
- 在云主机运行 `sudo DOMAIN=<正式域名> EMAIL=<运维邮箱> APP_PORT=<端口> bash deploy/bootstrap_https.sh`，并验证 `certbot renew --dry-run`。
- 生产 `.env.production` 必须设置：
  - `REDIS_PASSWORD`：生产随机长密码。
  - `METRICS_TOKEN`：Prometheus/SLS 采集 token。
  - `SENTRY_DSN`：若 `SENTRY_REQUIRED=1`。
  - `BACKUP_ENCRYPTION_RECIPIENT`：可解密的 GPG recipient。
  - `BACKUP_OFFSITE_TARGET`：真实异地位置，例如 rclone remote、对象存储或挂载的异地 NAS。
- 备份验收必须包含一次创建、一次异地可见性检查、一次解密、一次恢复演练。
- 监控验收必须至少覆盖：HTTP 5xx、429 激增、Redis 不可达、SQLite WAL 增长、磁盘容量、backup 失败、housekeeping 退出、worker dead-letter 堆积。

## 本轮补充修正

- `backend-housekeeping` 不再继承 Web 容器 HTTP 健康检查，改为成功执行后的心跳文件健康检查。
- `/api/metrics` 会记录自身 scrape 并输出进程信息，避免多 worker 场景下偶发只有 HELP 行。
- 备份脚本关闭 SQLite 连接后再替换临时文件，避免 Windows 本地 QA 中的文件占用问题。

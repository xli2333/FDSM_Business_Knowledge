# 06 · 高并发改造：AI 异步任务队列

> **前置依赖**：[04_鉴权加缓存与本地用户绑定](./04_代码改造_鉴权加缓存与本地用户绑定.md) 完成；Redis 可达（本地用 Docker 起一个即可）
> **预计耗时**：4-6 小时
> **完成标志**：
> - 编辑页点"AI 摘要" → 前端立刻返回 `task_id`，后台 worker 异步执行
> - 前端每 2 秒轮询 `GET /api/editorial/ai/tasks/{task_id}`，60 秒内能拿到结果
> - 同时 5 个用户触发 AI，web worker 不阻塞（`/api/home/feed` 还能正常响应）
> - worker 容器独立运行，处理能力不影响 web

---

## 背景与目标

### 为什么必须做这个

项目里 AI 调用是**同步阻塞**的，单次耗时 10-90 秒。调用链：

```
前端 → /api/editorial/ai/summarize → editorial_service.generate_summary
    → ai_service._invoke_prompt → requests.post(Gemini API, timeout=90)
    ← 阻塞等待 30 秒
    ← 返回结果
```

问题：
- **gunicorn 4 worker 被占满**：4 个用户同时点 "AI 摘要"，所有 worker 都卡住 30 秒
- **期间其他请求排队**：`/api/home/feed`、`/api/articles/*` 都等不到 worker 处理 → 502 Bad Gateway
- **容错差**：任何一个 Gemini 超时就是 90 秒 timeout，期间这个 worker 完全不可用
- **用户体验差**：前端转圈 30 秒等一个 HTTP 响应，断网后结果就丢了

### 解决思路

把 AI 调用从**同步 RPC 模式**改成**异步任务模式**：

```
前端 → POST /api/editorial/ai/summarize {article_id}
       ← 立刻返回 {task_id: "xxx", status: "pending"}   (< 50 ms)

       （web worker 释放，处理其他请求）

前端 → GET /api/editorial/ai/tasks/xxx   每 2 秒轮询
       ← {status: "pending"} → {status: "running"} → {status: "done", result: "..."}

backend-worker 容器独立拉队列、调 Gemini、写结果回 Redis
```

### 技术选型：为什么不用 Celery

| 方案 | 优点 | 缺点 | 本项目选择 |
|---|---|---|---|
| **Redis + 自建 loop**（本方案）| 简单、无新依赖、透明 | 无分布式调度、重试要手写 | ✅ v1 用这个 |
| Celery | 功能全、社区成熟 | 配置复杂、多一层 broker 抽象、有坑 | v2 再引入（看规模） |
| FastAPI BackgroundTasks | 零依赖 | 任务跟 web 进程共存亡，重启就丢 | 不够 |
| arq | 轻量 async | 生态小 | 可作为 Celery 备选 |

**核心判断**：项目 AI 调用量日估 100-500 次，完全在 Redis + 自建 loop 的能力范围内。不需要分布式调度。

### 步骤 2 · 新建 `backend/services/ai_task_service.py`

**创建文件 `backend/services/ai_task_service.py`**：

```python
"""
AI 异步任务服务。所有需要长时间处理的操作（Gemini 摘要、翻译、格式化、
RAG 文档摄入）都通过这里排队，由 backend-worker 容器消费。

任务数据结构（存 Redis key: aitask:{task_id}）：
    {
        "id": "<uuid>",
        "type": "editorial.summarize",
        "status": "pending" | "running" | "done" | "failed",
        "payload": {...},           # 调用参数
        "result": {...} | None,     # 成功时的结果
        "error": "<str>" | None,    # 失败时的错误信息
        "actor_user_id": "<str>",   # 发起人，用于权限校验和审计
        "created_at": <epoch>,
        "started_at": <epoch> | None,
        "finished_at": <epoch> | None,
    }

队列（Redis list key: aitask:queue）：LPUSH 入队，BRPOP 消费，FIFO 顺序。
"""
from __future__ import annotations

import json
import os
import time
import uuid
from typing import Any

import redis

# ========================================================================
# 客户端
# ========================================================================

_REDIS_URL = os.getenv("REDIS_URL", "redis://127.0.0.1:6379/0")

# 使用连接池。在多 worker 环境下每个进程独立一份，这是期望行为。
_REDIS: redis.Redis | None = None


def get_redis() -> redis.Redis:
    global _REDIS
    if _REDIS is None:
        _REDIS = redis.from_url(_REDIS_URL, decode_responses=True, socket_timeout=5)
    return _REDIS


# ========================================================================
# 常量
# ========================================================================

TASK_TTL_SECONDS = 3600                 # 任务结果保留 1 小时
QUEUE_KEY = "aitask:queue"
TASK_KEY_PREFIX = "aitask:"

VALID_TYPES = {
    "editorial.summarize",              # 生成文章摘要
    "editorial.translate",              # 中译英或反向
    "editorial.format",                 # AI 排版美化
    "editorial.tag_suggest",            # 标签建议
    "rag.ingest_article",               # RAG 文档摄入（向量化+索引）
    "daily_bookmark.refresh",           # 每日书签主题生成
}


# ========================================================================
# 公共 API
# ========================================================================

def create_task(task_type: str, payload: dict[str, Any], *, actor_user_id: str | None = None) -> str:
    """创建任务，LPUSH 到队列。返回 task_id。"""
    if task_type not in VALID_TYPES:
        raise ValueError(f"unknown task type: {task_type}")

    task_id = uuid.uuid4().hex
    task = {
        "id": task_id,
        "type": task_type,
        "status": "pending",
        "payload": payload,
        "result": None,
        "error": None,
        "actor_user_id": actor_user_id,
        "created_at": time.time(),
        "started_at": None,
        "finished_at": None,
    }
    r = get_redis()
    pipe = r.pipeline()
    pipe.setex(f"{TASK_KEY_PREFIX}{task_id}", TASK_TTL_SECONDS, json.dumps(task))
    pipe.lpush(QUEUE_KEY, task_id)
    pipe.execute()
    return task_id


def get_task(task_id: str) -> dict[str, Any] | None:
    """查询任务状态。返回 None 表示不存在或已过 TTL。"""
    r = get_redis()
    raw = r.get(f"{TASK_KEY_PREFIX}{task_id}")
    return json.loads(raw) if raw else None


def update_task(task_id: str, patch: dict[str, Any]) -> dict[str, Any] | None:
    """更新任务字段（shallow merge）。返回更新后的完整 task，不存在则返回 None。"""
    r = get_redis()
    raw = r.get(f"{TASK_KEY_PREFIX}{task_id}")
    if not raw:
        return None
    task = json.loads(raw)
    task.update(patch)
    r.setex(f"{TASK_KEY_PREFIX}{task_id}", TASK_TTL_SECONDS, json.dumps(task))
    return task


def pop_task(timeout: int = 5) -> tuple[str, dict[str, Any]] | None:
    """阻塞等待队列出一个任务。返回 (task_id, task_dict) 或 None（超时）。"""
    r = get_redis()
    # BRPOP 返回 [queue_key, value] 或 None
    result = r.brpop([QUEUE_KEY], timeout=timeout)
    if not result:
        return None
    _, task_id = result
    task = get_task(task_id)
    if not task:
        # 任务 TTL 内出队但 key 过期了（罕见），跳过
        return None
    return task_id, task


def queue_length() -> int:
    return get_redis().llen(QUEUE_KEY)


def cancel_task(task_id: str) -> bool:
    """标记任务为 failed（取消）。worker 消费时会跳过。返回是否成功。"""
    task = get_task(task_id)
    if not task:
        return False
    if task["status"] in ("done", "failed"):
        return False
    update_task(task_id, {
        "status": "failed",
        "error": "cancelled by user",
        "finished_at": time.time(),
    })
    return True
```

**设计点**：
- 用 Redis pipeline 原子化 `setex + lpush`，避免任务写了但没入队的情况
- 任务 TTL 1 小时：结果保留够前端轮询，也防止长期积累
- `VALID_TYPES` 白名单：防止调用方乱传 type
- `actor_user_id` 做审计：谁发起的任务可追溯

```python
@router.get("/ai/tasks/{task_id}")
def poll_task(task_id: str, authorization: str | None = Header(default=None)):
    """轮询任务状态。"""
    user = get_authenticated_user(authorization)
    if not user:
        raise HTTPException(401, "unauthorized")
    task = get_task(task_id)
    if not task:
        raise HTTPException(404, "task not found or expired")
    # 安全：非 admin 只能看自己发起的任务
    if not _is_admin(user) and task.get("actor_user_id") != user["id"]:
        raise HTTPException(403, "forbidden")
    return task


@router.delete("/ai/tasks/{task_id}")
def cancel(task_id: str, authorization: str | None = Header(default=None)):
    user = get_authenticated_user(authorization)
    if not user:
        raise HTTPException(401, "unauthorized")
    task = get_task(task_id)
    if not task:
        raise HTTPException(404)
    if not _is_admin(user) and task.get("actor_user_id") != user["id"]:
        raise HTTPException(403)
    ok = cancel_task(task_id)
    return {"cancelled": ok}
```

### 步骤 4 · 新建 `deploy/worker_loop.py`

**创建文件 `deploy/worker_loop.py`**：

```python
"""
Backend Worker 主循环。

运行环境：backend-worker 容器（docker-compose.yml 里定义）
命令：python -m deploy.worker_loop
职责：
  1. 消费 Redis 队列 aitask:queue 里的 AI 任务
  2. 定时 flush 浏览计数（见 §07）
  3. 每日生成书签主题

这个文件不挂在 FastAPI 里，是独立进程。因此它需要自己触发 WAL/表迁移。
"""
from __future__ import annotations

import logging
import os
import signal
import sys
import time
import traceback
from datetime import datetime

# --------------------------------------------------------------------
# 日志
# --------------------------------------------------------------------
logging.basicConfig(
    level=os.getenv("WORKER_LOG_LEVEL", "INFO"),
    format="%(asctime)s [%(levelname)s] %(name)s: %(message)s",
)
log = logging.getLogger("worker")


# --------------------------------------------------------------------
# 启动初始化
# --------------------------------------------------------------------
# 跟 web lifespan 同一套逻辑：让 worker 启动时也保证 DB 可用、表齐全
def _init():
    from backend.database import ensure_database_ready, ensure_runtime_tables
    ensure_database_ready()
    ensure_runtime_tables()
    log.info("worker initialized (pid=%s)", os.getpid())


# --------------------------------------------------------------------
# 任务处理器
# --------------------------------------------------------------------
def _handle_editorial_summarize(payload: dict) -> dict:
    from backend.services.editorial_service import generate_summary_for_task
    return generate_summary_for_task(**payload)


def _handle_editorial_translate(payload: dict) -> dict:
    from backend.services.editorial_service import translate_for_task
    return translate_for_task(**payload)


def _handle_editorial_format(payload: dict) -> dict:
    from backend.services.editorial_service import format_for_task
    return format_for_task(**payload)


def _handle_editorial_tag_suggest(payload: dict) -> dict:
    from backend.services.editorial_service import tag_suggest_for_task
    return tag_suggest_for_task(**payload)


def _handle_rag_ingest(payload: dict) -> dict:
    from backend.services.knowledge_ingestion_service import ingest_article_for_task
    return ingest_article_for_task(**payload)


def _handle_daily_bookmark_refresh(payload: dict) -> dict:
    from backend.services.daily_bookmark_service import refresh_daily_bookmarks
    refresh_daily_bookmarks(**payload)
    return {"status": "refreshed"}


DISPATCH_TABLE = {
    "editorial.summarize": _handle_editorial_summarize,
    "editorial.translate": _handle_editorial_translate,
    "editorial.format": _handle_editorial_format,
    "editorial.tag_suggest": _handle_editorial_tag_suggest,
    "rag.ingest_article": _handle_rag_ingest,
    "daily_bookmark.refresh": _handle_daily_bookmark_refresh,
}


def process_task(task_id: str, task: dict) -> None:
    """消费一个任务，更新状态到 Redis。异常安全——任何异常都会被 catch 并标记任务 failed。"""
    from backend.services.ai_task_service import update_task

    task_type = task.get("type")
    handler = DISPATCH_TABLE.get(task_type)
    if handler is None:
        log.error("task %s: unknown type %s", task_id, task_type)
        update_task(task_id, {
            "status": "failed",
            "error": f"unknown task type: {task_type}",
            "finished_at": time.time(),
        })
        return

    log.info("task %s: start type=%s", task_id, task_type)
    update_task(task_id, {
        "status": "running",
        "started_at": time.time(),
    })

    t0 = time.time()
    try:
        result = handler(task.get("payload") or {})
    except Exception as exc:
        log.exception("task %s failed", task_id)
        update_task(task_id, {
            "status": "failed",
            "error": f"{type(exc).__name__}: {exc}",
            "finished_at": time.time(),
        })
        return

    elapsed = time.time() - t0
    log.info("task %s: done in %.1fs", task_id, elapsed)
    update_task(task_id, {
        "status": "done",
        "result": result,
        "finished_at": time.time(),
    })


# --------------------------------------------------------------------
# 周期任务（占位，§07 会填充 flush_view_counts）
# --------------------------------------------------------------------
def flush_view_counts() -> None:
    """占位：§07 里实现。"""
    pass


def run_daily_bookmark_if_due() -> None:
    """每天 03:00 自动触发一次每日书签生成（如果今天还没生成过）。"""
    from backend.services.ai_task_service import get_redis
    from backend.services.daily_bookmark_service import refresh_daily_bookmarks

    now = datetime.now()
    if now.hour != 3:    # 只在 03:00-03:59 之间执行
        return

    today = now.date().isoformat()
    key = f"daily_bookmark:done:{today}"
    r = get_redis()
    # SET NX EX：只设一次，24 小时后自动过期
    if r.set(key, "1", nx=True, ex=86400):
        log.info("triggering daily bookmark for %s", today)
        try:
            refresh_daily_bookmarks()
        except Exception:
            log.exception("daily bookmark failed")
            r.delete(key)    # 失败了让下小时再试


# --------------------------------------------------------------------
# 信号处理
# --------------------------------------------------------------------
_shutdown = False


def _handle_signal(signum, frame):
    global _shutdown
    log.info("received signal %s, shutting down gracefully", signum)
    _shutdown = True


signal.signal(signal.SIGTERM, _handle_signal)
signal.signal(signal.SIGINT, _handle_signal)


# --------------------------------------------------------------------
# 主循环
# --------------------------------------------------------------------
def main():
    _init()

    from backend.services.ai_task_service import pop_task

    log.info("worker main loop started")
    iteration = 0
    while not _shutdown:
        iteration += 1
        try:
            # 1. 优先消费一个 AI 任务（最多阻塞 5 秒）
            result = pop_task(timeout=5)
            if result is not None:
                task_id, task = result
                # 已被 cancel 的任务跳过
                if task.get("status") == "failed" and task.get("error") == "cancelled by user":
                    log.info("skipping cancelled task %s", task_id)
                else:
                    process_task(task_id, task)
                continue

            # 2. 没任务，做周期事情
            if iteration % 6 == 0:          # 每 ~30 秒（6 × 5s timeout）
                flush_view_counts()

            if iteration % 60 == 0:         # 每 ~5 分钟
                run_daily_bookmark_if_due()

        except Exception:
            log.exception("worker iteration %d failed", iteration)
            time.sleep(5)

    log.info("worker exiting")


if __name__ == "__main__":
    try:
        main()
    except Exception:
        log.exception("worker crashed")
        sys.exit(1)
```

**关键设计**：
- 模块顶层**不 import backend**，只在函数内部 import。原因：某些周期任务和某些 handler 不需要加载全部业务模块，按需 import 降低启动内存
- 信号处理：收到 SIGTERM 时设 `_shutdown=True`，当前任务做完后主循环退出。Docker `stop` 的 10 秒 grace period 够用
- `pop_task(timeout=5)`：没任务时每 5 秒醒一次做周期事情，这样周期任务延迟最多 5 秒
- 一次只处理一个任务（worker 容器本身可以多副本）

### 步骤 5 · 给 editorial_service 加"任务版"函数

worker 调用的 handler 函数签名必须是 `(payload_dict) → result_dict`。`editorial_service` 里现有的同步函数签名可能不是这样，需要薄包装。

### 步骤 6 · 前端改轮询

**打开 `frontend/src/pages/EditorialWorkbenchPage.jsx`**（或对应触发 AI 的组件）。

**搜索现有的调用**（大概长这样）：
```javascript
async function handleSummarize() {
  setLoading(true)
  try {
    const result = await api.summarize({ article_id: id })
    setSummary(result.summary_markdown)
  } finally {
    setLoading(false)
  }
}
```

**改成异步+轮询**：
```javascript
async function handleSummarizeAsync(articleId) {
  setStatus('queued')
  try {
    // 1. 发起异步任务
    const { task_id } = await api.enqueueSummarize({ article_id: articleId })

    // 2. 轮询结果，每 2 秒一次，最多 120 秒
    const result = await pollTask(task_id, {
      intervalMs: 2000,
      timeoutMs: 120000,
      onProgress: (task) => setStatus(task.status),
    })

    // 3. 拿到结果更新 UI
    setSummary(result.result?.summary_markdown || '')
    setStatus('done')
  } catch (err) {
    setStatus('failed')
    setErrorMsg(err.message)
  }
}

async function pollTask(taskId, { intervalMs = 2000, timeoutMs = 120000, onProgress } = {}) {
  const deadline = Date.now() + timeoutMs
  while (Date.now() < deadline) {
    const task = await api.pollTask(taskId)
    if (onProgress) onProgress(task)
    if (task.status === 'done') return task
    if (task.status === 'failed') {
      throw new Error(task.error || 'task failed')
    }
    await new Promise(r => setTimeout(r, intervalMs))
  }
  throw new Error('task polling timeout')
}
```

**在 `frontend/src/api/index.js` 新增 API 包装函数**：
```javascript
export function enqueueSummarize(body, opts = {}) {
  return request('/editorial/ai/tasks/summarize', {
    method: 'POST',
    body: JSON.stringify(body),
    ...opts,
  })
}

export function enqueueTranslate(body, opts = {}) {
  return request('/editorial/ai/tasks/translate', {
    method: 'POST',
    body: JSON.stringify(body),
    ...opts,
  })
}

export function enqueueFormat(body, opts = {}) {
  return request('/editorial/ai/tasks/format', {
    method: 'POST',
    body: JSON.stringify(body),
    ...opts,
  })
}

export function pollTask(taskId, opts = {}) {
  return request(`/editorial/ai/tasks/${encodeURIComponent(taskId)}`, opts)
}

export function cancelTask(taskId, opts = {}) {
  return request(`/editorial/ai/tasks/${encodeURIComponent(taskId)}`, {
    method: 'DELETE',
    ...opts,
  })
}
```

**UI 状态设计**（参考）：
- `idle`：没在跑
- `queued`：已入队，等 worker
- `running`：worker 在处理
- `done`：完成，显示结果
- `failed`：失败，显示错误

按钮文案随状态变：
- idle → "生成摘要"
- queued / running → "生成中...(可关页面，稍后回来看结果)"
- done → 显示结果 + "重新生成"
- failed → 显示错误 + "重试"

## 常见错误与排查

| 症状 | 原因 | 修复 |
|---|---|---|
| `redis.exceptions.ConnectionError` | Redis 没起或端口错 | `docker start fdsm-redis-dev`；检查 `REDIS_URL` |
| Worker 启动立即退出 | `_init()` 抛异常 | 看 stderr，多半是 DB 路径错 |
| `unknown task type` | 打 handler table 时漏了 | 检查 `VALID_TYPES` 和 `DISPATCH_TABLE` 键对齐 |
| `_handle_editorial_summarize` 报 ImportError | `editorial_service` 里没对应函数 | 步骤 5 里改对实际函数名 |
| 前端轮询返回 404 | 任务 TTL 过了 | 1 小时内没拿走就被 Redis 清了，让前端 timeout 更短 |
| 前端轮询 403 | 用户不是 actor 且不是 admin | 权限设计如此，换 admin 身份或让发起者自己看 |
| worker 日志看到 `cancelled task` 但任务其实在跑 | cancel 只标记状态，worker handler 没中断 | 长任务中断需要线程 cancel；v1 简化为"标记 + 结果丢弃" |
| 并发压测时 Redis OOM | `maxmemory` 限太小 | Redis 启动时加 `--maxmemory 1gb --maxmemory-policy allkeys-lru` |

## 边界与权衡

**为什么取消不真的中断任务**：
Python 没有安全的"强制中断其他函数"的机制。真要做，得把 handler 写成异步 + 每隔几秒自查 `cancel` 标志位。v1 简化为"cancel 仅标记，worker 跑完会看到 cancelled 状态并丢弃结果"。

**为什么不用 Redis Stream 而用 List**：
Redis List + BRPOP 是最简 FIFO 队列。Stream 多了消费组、ack 机制（功能好但复杂）。v1 不需要。

**TTL 1 小时够吗**：
够。前端轮询 2 分钟超时。任务结果 1 小时内不访问可以重新发起（幂等）。

**Worker 副本数**：
`docker-compose.yml` 里 `backend-worker` 默认 1 副本。如果 AI 任务堆积，起 2 副本即可（`docker compose up -d --scale backend-worker=2`）。Redis 队列天然支持多消费者竞争。

# Round11 访客验证与用户互动体系补齐

## 本轮目标

- [x] 盘点当前知识库在访客识别、用户登录、互动沉淀与内容热度层面的商业化缺口
- [x] 设计匿名访客与登录用户并存的产品方案，明确浏览、点赞、收藏的权限边界
- [x] 接入 Supabase Auth，支持前端用户登录状态读取、邮箱 OTP 登录与会话持久化
- [x] 在后端补齐访客与用户行为数据结构，支持阅读事件、点赞、收藏与聚合统计
- [x] 将文章浏览量从单纯打开即加一，升级为按访客去重的真实阅读事件统计
- [x] 新增文章互动接口与用户状态接口，支持点赞、收藏、取消与个人状态读取
- [x] 在前端文章页与卡片体系中增加点赞、收藏、登录入口与热度展示
- [x] 在首页或热点模块中引入基于真实阅读和互动的热度排序依据
- [x] 扩展后台烟雾测试，覆盖匿名浏览、登录态占位校验、点赞收藏与统计接口
- [x] 扩展前端视觉验收，覆盖文章互动与登录入口渲染
- [x] 运行编译、构建、烟雾测试与视觉验收，确保无中文编码问题与 JSX 问题

## 产品约束

- [x] 未登录访客可以浏览和产生浏览事件，但不能点赞或收藏
- [x] 已登录用户可以点赞、收藏，并可看到自己的互动状态
- [x] 浏览量需要具备基础去重逻辑，避免连续刷新被重复记为高热文章
- [x] Supabase 仅承担用户身份与会话控制，不改变现有业务文章数据来源范围

## 验收标准

- [x] 站点存在可工作的登录入口和登录状态展示
- [x] 文章页能显示真实浏览量、点赞数、收藏数与当前用户状态
- [x] 未登录点击点赞或收藏会得到明确引导
- [x] 已登录用户可以完成点赞和收藏切换
- [x] 首页或热榜能体现新的互动热度逻辑
- [x] `python -m compileall backend` 通过
- [x] `python backend/scripts/smoke_test.py` 通过
- [x] `npm run lint` 通过
- [x] `npm run build` 通过
- [x] `npm run visual:acceptance` 通过

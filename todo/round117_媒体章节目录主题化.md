# Round 117：媒体章节目录主题化

## 对齐结论
- [x] 当前媒体章节已经能识别时间戳，但标题仍偏“原话摘句”，更像说了哪一句，不像目录在概括这一部分讲什么。
- [x] 本轮目标是把媒体章节标题收口成“主题化目录”，优先按整段内容概括出每一部分的核心议题，而不是直接截取开头原句。
- [x] 本轮继续保持轻量实现，不新增复杂章节编辑器；重点放在后端章节标题生成策略升级，并让前端直接复用结果。
- [x] 本轮真实验收会覆盖更自然的中文转录段落，避免目录再次退化成“欢迎来到”“看完之后”这类口语起手句。
- [x] 全部 todo 文档与代码改动保持 UTF-8 中文，不引入中文编码错误或 JSX / Python 语法错误。

## 原子任务
- [x] P0 | 后端目录 | 重构媒体章节标题提取逻辑，按完整章节段落生成主题化标题，而不是只取第一句原话。
- [x] P0 | 后端规则 | 增加口语开场、衔接语、噪声句的剔除与降权逻辑，避免 `欢迎来到`、`看完之后`、`不过随着` 这类起手句直接进入目录。
- [x] P0 | 后端命名 | 为主题化标题补轻量前缀策略，例如“问题引入 / 核心悬念 / 招股书拆解 / 行业背景 / 资本路径”，让目录更像结构化提纲。
- [x] P0 | 测试 | 更新后端测试，覆盖真实中文转录格式下生成出的章节标题是“讲什么”，而不是“说了什么话”。
- [x] P0 | 验收 | 更新 `frontend/scripts/media_acceptance_round106.mjs`，让真实验收样本更接近自然中文播客转录，并校验目录不会退化成口语原句。
- [x] P0 | 验证 | 跑通 `pytest backend/tests/test_admin_editorial_media_features.py -q`、`npm run build`、`npm run media:acceptance:round106`。

## 结束标准
- [x] 章节标题能更稳定地概括每一部分在讲什么，而不是摘录第一句原话。
- [x] 常见口语开场和承接句不会直接成为目录标题。
- [x] 中文真实转录样本生成出的目录更像文章提纲。
- [x] `pytest backend/tests/test_admin_editorial_media_features.py -q`、`npm run build`、`npm run media:acceptance:round106` 通过。

## 备注

# Round37 首页首屏精选收紧与信息补位

## 目标

解决首页首屏两个明显问题：
- 右侧“编辑精选”列表过长，压住首屏节奏
- 左侧搜索框下方留白过大，信息密度不够
- 在不新增后端接口的前提下，复用现有首页数据完成信息补位
- 保持中英文双语与移动端布局稳定，不引入中文编码异常

## 原子任务

- [x] P0 | 首屏结构 | 复核首页首屏左右列的高度失衡点
- [x] P0 | 编辑精选 | 收紧编辑精选数量、卡片高度与摘要长度
- [x] P0 | 搜索区 | 在搜索框下复用现有 `hot_tags/topics` 数据补充紧凑信息层
- [x] P1 | 信息层级 | 移除首页内重复的热门标签展示，避免同页重复
- [x] P1 | 验收 | 运行前端构建并检查是否存在编码或编译问题
- [x] P1 | 回填 | 按完成情况逐项勾选并记录本轮结论

## 本轮结论

- 已收紧首页首屏右侧“编辑精选”，从 4 条降为 3 条，并对标题与摘要做截断控制
- 已在搜索框下补入紧凑信息层，复用现有热门标签与专题数据，消除首屏左侧大面积留白
- 已移除原页面后续重复的“热门标签”整块，避免同页重复表达
- 已通过 `npm.cmd run build` 与 `npm.cmd run lint`
- 基于本轮目标与现有首页数据结构，首屏的明确失衡点已经处理完成；除非后续要新增个性化推荐、真实用户分层或新的视觉物料，否则没有足够明确、需要立刻再开一轮的改进项

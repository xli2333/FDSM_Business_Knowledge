# Round52 账单方案表与配置价格同步修复
## 目标

解决“配置里的会员价格已改，但页面仍显示旧价格”的数据同步问题。

## 原子任务

- [x] P0 | 根因定位 | 确认会员方案接口读取的是 `billing_plans` 表而不是直接读取配置
- [x] P0 | 启动同步 | 修复运行时方案同步逻辑，让 `billing_plans` 按 `config` 自动 upsert
- [x] P0 | 立即修正 | 直接把当前数据库中的方案价格同步到最新配置
- [x] P1 | 验收 | 校验接口数据源与本地价格记录已变为月付 `30`、年付 `300`
- [x] P1 | 回填 | 记录本轮结论

## 本轮结论

- 根因不是前端没改，而是后端接口读取数据库中的旧方案记录
- 现在启动时会自动把 `billing_plans` 与配置对齐，避免后续再出现“代码改了但页面没变”
- 当前数据库中的付费月订阅和付费年订阅也已同步到最新价格
- 本机运行中的 `/api/billing/plans` 已实测返回月付 `3000` 分、年付 `30000` 分

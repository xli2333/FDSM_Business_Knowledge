# Round 122：媒体正式详情页与流页拆分

## 对齐结论
- [x] 当前音频与视频只有 `/audio`、`/video` 流式列表页，没有像文章 `/article/:id` 那样的正式详情页。
- [x] 本轮目标是给每条已发布音频 / 视频建立正式详情页，列表页只承担“发现与进入详情”的职责，不再承担完整详情展示。
- [x] 媒体正式详情页按文章页思路收口：能看到标题、封面、发布时间、时长、摘要、节目简介、章节目录与播放器。
- [x] 详情页继续保留会员权限规则：公开内容可完整播放，受限内容按试看 / 试听规则收口。
- [x] 管理员在正式详情页仍可见“重新编辑”入口，点击后回到草稿箱，不回后台列表找稿。
- [x] 路由按现有媒体分类拆开：音频走 `/audio/:slug`，视频走 `/video/:slug`。
- [x] 全部 todo 与代码改动保持 UTF-8 中文，不引入中文编码错误或 JSX / Python 语法错误。

## 原子任务
- [x] P0 | Todo | 新建本轮中文 todo，并按原子项推进与勾选。
- [x] P0 | 后端详情接口 | 新增公开媒体详情接口，支持按 `kind + slug` 获取单条已发布媒体详情。
- [x] P0 | 后端序列化 | 补齐详情页所需字段，明确返回摘要、节目简介、章节、播放器地址、试看权限、封面、系列、主讲人等。
- [x] P0 | 前端路由 | 在前端新增 `/audio/:slug` 与 `/video/:slug` 路由，不影响现有 `/audio` 与 `/video` 流页。
- [x] P0 | 前端详情页 | 新建媒体详情页组件，按文章页思路组织页面结构，展示标题、摘要、简介、目录、播放器与权限提示。
- [x] P0 | 前端目录交互 | 章节目录支持点击跳转到对应播放时间点；视频与音频都复用同一套目录跳播逻辑，并在 UI 上同步高亮当前章节。
- [x] P0 | 前端流页收口 | `/audio` 与 `/video` 卡片改为“展示摘要 + 进入详情页”入口，不再在流页卡片中承载完整简介和大段正文。
- [x] P0 | 前端管理员入口 | 在媒体详情页提供管理员专属“重新编辑”按钮，并保持已发布内容从正式页回到草稿箱的既有规则。
- [x] P0 | 前端链接统一 | 媒体卡片标题、封面、主按钮统一跳详情页；原“打开完整音频 / 视频”语义收口到详情页。
- [x] P0 | 样式布局 | 详情页视觉对齐文章正式页，但保留媒体播放器、章节目录和权限提示的媒体特性，不做成纯文章模板复制。
- [x] P0 | 测试 | 补后端测试，覆盖公开媒体详情、受限媒体详情、slug 路由、章节字段与管理员回编入口。
- [x] P0 | 验收 | 更新真实验收脚本，覆盖“流页进入详情页 -> 查看摘要/简介/章节 -> 点击目录跳播 -> 管理员重新编辑”。
- [x] P0 | 验证 | 复跑 `pytest backend/tests/test_admin_editorial_media_features.py -q`、`npm run build`、`npm run media:acceptance:round106`。

## 结束标准
- [x] 每条已发布音频 / 视频都有可访问的正式详情页。
- [x] `/audio` 与 `/video` 流页只负责浏览和进入详情，不再承担完整详情信息。
- [x] 媒体详情页能稳定展示摘要、节目简介和章节目录。
- [x] 章节目录点击后能驱动播放器跳到对应时间点。
- [x] 管理员可直接从正式详情页进入重新编辑。
- [x] `pytest backend/tests/test_admin_editorial_media_features.py -q`、`npm run build`、`npm run media:acceptance:round106` 通过。

## 备注
- [x] 本轮只新增媒体正式详情页链路，不重做媒体数据模型，也不扩展人工章节编辑器。

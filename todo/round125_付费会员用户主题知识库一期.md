# Round 125：付费会员用户主题知识库一期

## 对齐结论
- [x] `stock_portfolio` 的知识库模式已经确认是“用户私有资料池 + 选定内容喂给 AI + Markdown 渲染”，本轮把这套模式迁移为复旦知识库里的“用户自建主题知识库”。
- [x] 本轮知识库只收录站内文章，不先扩展到音频、视频或外部 PDF 上传。
- [x] 用户可以创建多个私有主题并自行命名，一篇文章可以加入多个用户主题。
- [x] “用户主题知识库”只对付费会员与管理员开放；未登录用户和免费会员只会看到登录或升级引导。
- [x] 文章详情页增加“加入知识库”入口；如果用户没有主题或只有一个主题，会明确提示先创建收藏主题，并支持直接新建主题后把当前文章加入进去。
- [x] 知识库页面视觉参考 `stock_portfolio` 的卡片流式工作台，但整体配色、按钮、留白和层次全部遵守复旦知识库现有设计语言。
- [x] 主题详情页包含：主题信息、收录文章卡片流、主题内 AI 助理，且 AI 回答仅基于该主题已收录文章。
- [x] 全部 todo 与代码改动保持 UTF-8 中文，不引入中文编码错误或 JSX / Python 语法错误。

## 原子任务
- [x] P0 | Todo | 新建本轮中文 todo，并按原子项推进与勾选。
- [x] P0 | 数据层 | 新建用户主题表与主题文章关联表，按 `user_id` 做私有隔离并建立索引。
- [x] P0 | 权限层 | 新增付费会员 / 管理员专用的用户主题知识库权限校验。
- [x] P0 | 后端主题接口 | 支持主题创建、重命名、删除、列表、详情，以及文章加入 / 移出主题。
- [x] P0 | 后端检索 | 为现有站内检索增加“限定 `article_ids` 范围”的能力，支持主题内检索。
- [x] P0 | 后端 AI | 新增主题内 AI 问答接口，只从该主题收录文章生成回答与引用。
- [x] P0 | 文章页入口 | 在文章详情页收藏区增加“加入知识库”入口，并接入主题选择 / 新建弹层。
- [x] P0 | 创建主题引导 | 当用户没有主题或仅有一个主题时，在加入流程里强化“先创建收藏主题”的提示和新建入口。
- [x] P0 | 前端知识库总页 | 新建 `/me/knowledge`，展示我的主题卡片流、创建入口和会员状态提示。
- [x] P0 | 前端主题详情页 | 新建 `/me/knowledge/:slug`，展示主题头部、文章卡片流与主题内 AI 助理工作台。
- [x] P0 | 设计语言 | 页面风格对齐 `stock_portfolio` 知识库的双栏工作台与卡片流，但使用复旦知识库的蓝橙配色、圆角、阴影和排版。
- [x] P0 | 导航入口 | 在已登录付费会员路径里补“我的知识库”入口，确保用户能稳定回到知识库系统。
- [x] P0 | 测试 | 补后端测试，覆盖用户隔离、付费权限、创建主题、文章加入主题、主题内 AI 检索范围。
- [x] P0 | 验收 | 新增真实验收脚本，覆盖“文章页创建主题并加入 -> 知识库页查看卡片流 -> 主题内 AI 提问”的完整路径。
- [x] P0 | 验证 | 复跑 `pytest` 定向测试、`npm run build` 与新知识库验收脚本，完成视觉截图抽检。

## 结束标准
- [x] 付费会员与管理员可以创建自己的私有知识库主题，并把文章加入其中。
- [x] 文章页“加入知识库”流程在零主题 / 单主题场景下会明确提示先创建收藏主题。
- [x] `/me/knowledge` 与 `/me/knowledge/:slug` 已上线，页面为复旦知识库风格的卡片流工作台。
- [x] 主题内 AI 助理回答仅基于当前主题已收录文章，不串用全站内容。
- [x] 未登录用户和免费会员无法进入该系统，只会得到登录或升级引导。
- [x] `pytest backend/tests/test_user_knowledge_features.py -q`、`npm run build`、`npm run knowledge:acceptance:round125` 通过。

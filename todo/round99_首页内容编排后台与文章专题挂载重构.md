# Round 99：首页内容编排后台与文章专题挂载重构

## 对齐结论
- [x] 文章后台要补“选专题”能力，编辑稿在发布前就能选择要进入哪些专题，发布时自动同步到正式专题文章关系。
- [x] 首页不再继续依赖“部分硬编码 + 部分数据库默认排序”的混合方案，头条、编辑推荐、快速入口、专题起读、栏目导航、专题广场都改为后台可配置。
- [x] 热门文章不再只给一个固定榜单，改为“日榜 / 周榜 / 月榜”三档切换，按浏览、点赞、收藏加权计算。
- [x] 最新文章仍然保持自动按发布时间倒序，不做人工挑选；首版目标是“刷新首页或重新请求接口即可看到最新结果”，不做 WebSocket 式实时推送。
- [x] 本轮先做一套新的“内容运营后台”，让管理员决定首页放什么文章、什么标签、什么专题、什么栏目。

## 当前现状核对
- [x] 文章发布链路当前只会同步 `article_tags` 和 `article_columns`，不会把编辑稿选中的专题同步到 `topic_articles`。
- [x] 首页头条和编辑推荐当前来自 `featured_articles`，但没有独立后台可维护。
- [x] 首页快速入口当前直接取高频标签，不是后台指定标签。
- [x] 首页专题起读当前直接取专题列表前两个，不是后台指定专题。
- [x] 首页栏目导航当前依赖 `COLUMN_DEFINITIONS` 常量，不是后台指定显示哪些栏目。
- [x] 首页专题广场当前直接取专题列表前四个，不是后台指定显示哪些专题。
- [x] 首页热门文章当前只有一个固定榜单，没有日 / 周 / 月切换，也没有后台权重设置。
- [x] 首页最新文章当前已经是 `publish_date DESC, id DESC`，但发布后能否在首页正确体现，取决于发布时是否正确落了栏目、专题和首页区块关系。

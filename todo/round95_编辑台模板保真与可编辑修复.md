# Round 95：编辑台模板保真与可编辑修复
## 对齐规则
- [x] 当前最高优先级不是加新功能，而是修复两个严重错误：原公众号模板被改坏、正文点击后光标立刻失效。
- [x] 编辑台右侧必须优先保留原公众号排版模板的真实视觉结构，不能再用新的通用正文样式覆盖原模板。
- [x] 编辑态必须能稳定编辑标题、正文、强调和表格内容，不能出现点击后立即失焦或选区被重置。
- [x] 保存后的最终 HTML 必须保留 `<style>`、表格结构、强调标签和模板元信息，不能再把模板样式在保存时剥掉。
- [x] 本轮必须补自动化测试，并做截图验收；至少覆盖标题、表格、强调、正文四类内容。
- [x] 全部代码与 todo 文案保持 UTF-8 中文，避免中文编码和 JSX 问题。

## 本轮完成
- [x] P0 | 排查 | 核实模板变形的根因包括：前端通用 `.tiptap` 样式覆盖原模板、手工保存时后端错误删除 `<style>` 与 `data-*`、自动排版默认剥离了原模板头部信息。
- [x] P0 | 排查 | 核实光标消失的根因是编辑器在父组件每次重渲染后反复 `setContent`，导致选区被重置。
- [x] P0 | 后端模板 | 调整公众号模板渲染输入，默认保留原模板头部信息，不再自动剥离标题/作者/编辑等头部区。
- [x] P0 | 后端保存 | 修正最终 HTML 清洗逻辑，继续去掉脚本与事件处理器，但保留 `<style>`、类名、`data-*`、表格和模板结构。
- [x] P0 | 后端兼容 | 为已存在的旧编辑稿增加“按已存 render_plan 恢复完整模板 HTML”的修复逻辑，避免旧稿继续以阉割版模板打开。
- [x] P0 | 前端编辑器 | 将右侧最终稿从会破坏模板结构的 TipTap 画布切回“基于原始 HTML 文档的 iframe 编辑态”，让编辑直接发生在真实模板上。
- [x] P0 | 前端编辑器 | 修复点击后光标消失问题，保证编辑 iframe 在输入时不会因为 React 重渲染被重建。
- [x] P0 | 前端编辑器 | 工具栏继续保留基础编辑能力：标题、正文、加粗、斜体、下划线、删除线、对齐、列表、引用、分隔线、颜色、撤销重做。
- [x] P0 | 前端编辑器 | 编辑态与预览态统一基于同一份完整 HTML，确保“所见即所得”，不再出现预览和编辑两套模板。
- [x] P0 | 数据链路 | 保存人工修改后继续写回完整 `final_html` 和 `editor_document`，刷新页面后模板与人工修改都能完整恢复。
- [x] P0 | 自动化测试 | 新增后端测试，覆盖完整 HTML 保存后标题、表格、强调、正文与样式块都不会丢失。
- [x] P0 | 自动化测试 | 新增后端测试，覆盖旧的裁剪版公众号 HTML 在详情读取时会按 render plan 自动恢复完整模板。
- [x] P0 | 截图验收 | 新增 Playwright 编辑台验收脚本，在临时数据库中验证真实模板截图、标题/正文/表格编辑、保存后刷新仍保留。

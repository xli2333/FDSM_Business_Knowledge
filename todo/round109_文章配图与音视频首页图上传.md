# Round 109：文章配图与音视频首页图上传

## 对齐结论
- [x] 文章编辑台需要补上真正的配图能力，不能只留 `cover_image_url` 文本字段。
- [x] 文章配图要同时支持本地上传与复制粘贴图片两种入口，并在草稿流里保存。
- [x] 音频与视频后台都需要首页图上传能力，但这轮只要求上传，不要求粘贴。
- [x] 文章发布后，上传的配图必须进入正式文章首页卡片与详情封面链路；媒体发布后，首页图必须进入媒体列表展示链路。

## 原子任务
- [x] P0 | 后端文章 | 新增文章封面图上传链路，支持把图片落盘并写回 `editorial_articles.cover_image_url`。
- [x] P0 | 后端文章 | 调整文章发布映射，让编辑台封面图进入正式文章卡片展示链路，而不是发布后丢失。
- [x] P0 | 后端媒体 | 给音频 / 视频上传接口增加首页图上传用途，并把结果写回媒体草稿与正式内容。
- [x] P0 | 前端文章 | 在编辑台增加配图上传区、图片预览与清空动作，不再只靠手填 URL。
- [x] P0 | 前端文章 | 在编辑台增加图片粘贴入口，允许管理员把剪贴板图片直接贴成文章配图。
- [x] P0 | 前端媒体 | 在媒体工作台增加首页图上传按钮、图片预览与发布链路同步。
- [x] P0 | 前端消费层 | 确认文章首页卡片 / 媒体列表会消费新的上传图片地址。
- [x] P0 | 测试 | 补后端测试，覆盖文章封面图上传发布、媒体首页图上传发布。
- [x] P0 | 验收 | 补前端验收脚本，覆盖文章配图上传 / 粘贴与媒体首页图上传，并重新跑构建和测试。

## 结束标准
- [x] 文章编辑台可以上传图片，也可以通过粘贴图片设置配图。
- [x] 文章发布后，首页 / 卡片 / 详情使用上传后的配图，不再丢失。
- [x] 音频与视频后台都可以上传首页图，并在媒体列表正常显示。
- [x] 后端测试、前端构建、真实验收全部通过。

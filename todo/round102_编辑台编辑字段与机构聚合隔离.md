# Round 102：编辑台编辑字段与机构聚合隔离

## 对齐结论
- [x] 编辑台顶部基础信息区里原有的“机构”输入在当前业务里实际被当作编辑署名使用，继续保留“机构”文案会误导录入。
- [x] 这列数据此前不仅影响编辑台显示，还会进入自动打标签信号和正式文章的机构聚合字段，导致“编辑”被误当成机构标签或机构页数据。
- [x] 本轮不能只改前端占位文案，必须同时切断编辑稿到标签系统、机构聚合、机构页链路的错误透传。
- [x] 本轮范围限定为“修复后续新稿与重发布链路”，不额外做历史正式文章的数据回填。

## 原子任务
- [x] P0 | 前端 | 将编辑台基础信息区该输入的中文占位从“机构”改为“编辑”，避免继续误导录入。
- [x] P0 | 前端 | 清空该字段的默认值，不再默认写入 `Fudan Business Knowledge` 这类机构名。
- [x] P0 | 后端 | 调整编辑稿自动打标签链路，不再把该字段作为实体/机构信号源参与标签抽取。
- [x] P0 | 后端 | 调整编辑稿栏目兜底推断，不再把该字段作为机构信号参与栏目判断。
- [x] P0 | 后端 | 调整编辑稿发布链路，不再把该字段写入正式文章 `primary_org_name` / `org_text` 等机构聚合字段，避免污染机构页与搜索。
- [x] P0 | 后端 | 保留该字段在编辑模板渲染里的编辑署名用途，不影响成品稿头部的编辑信息展示。
- [x] P0 | 验收 | `python -m py_compile backend/services/editorial_service.py`
- [x] P0 | 验收 | `npm run build`

## 结束标准
- [x] 编辑台该输入对管理员展示为“编辑”，不再显示“机构”。
- [x] 新建或修改编辑稿时，这列内容不会再被自动打成机构类标签或实体噪音标签。
- [x] 发布后的正式文章不会再因为该列内容生成错误的机构聚合数据。
- [x] 前后端改动均通过最小构建与语法验证。

## 备注
- [x] 历史上已经发布且被错误写入机构字段的文章不会被本轮自动修复；如需清理，需重新发布对应编辑稿。

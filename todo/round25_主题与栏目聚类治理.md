# Round25 主题与栏目聚类治理

## 本轮目标

- [x] 重新校准专题生成策略，避免当前“仅 8 个种子专题 + 默认不生成扩展专题”的半成品状态
- [x] 收紧栏目分发规则，降低当前栏目重叠过高与误入栏目问题
- [x] 为主题与栏目聚类补齐可量化验收指标，后续每轮都能直接复查质量
- [x] 重建数据库并抽样验收，确保聚类结果从规则到数据都一致

## 当前诊断结论

- [x] 当前专题总数仅 8 个，且全部来自预设 seed，未达到 `phase3_专题系统.md` 中 15+ 自动专题的目标
- [x] 当前 `rebuild_topics()` 默认 `limit_auto=0`，说明默认重建流程不会生成扩展自动专题
- [x] 当前自动专题候选偏弱，现有规则更容易生成 `案例教学专题`、`阿里巴巴专题`、`心理学专题` 这类局部标签专题，缺少更稳定的组合聚类
- [x] 当前栏目重叠偏高：平均每篇文章命中 2.1 个栏目，1492 篇文章命中 2 个及以上栏目，777 篇文章命中 3 个及以上栏目
- [x] 当前 `行业观察` 存在明显误入：已有 871 篇进入该栏目但没有任何 `industry` 类标签
- [x] 当前 `院长说` 规则过宽：仅因文章人物或机构包含复旦师资/学院信息，就可能被纳入，现状共有 629 篇，明显超出“院长说”应有边界

## 待解决问题

### 一、主题聚类

- [x] 将“seed 命中专题”与“自动聚类专题”明确分层，不再混用为同一种 `auto`
- [x] 引入基于高信号标签组合的候选主题，不再只按单标签扩展
- [x] 给自动专题增加低信号过滤，避免生成实体词或偶发热点词专题
- [x] 给专题增加质量阈值：文章数、时间跨度、标签纯度、文章重叠率
- [x] 重新定义专题上限与排序，优先保留高信号主线专题

### 二、栏目聚类

- [x] `行业观察` 建立真实行业命中门槛，不再允许无 `industry` 标签文章误入
- [x] `院长说` 收紧为“院长 / 教授 / 核心师资明确发声”而非泛复旦相关内容
- [x] `学术前沿` 继续以研究解读与学术成果为主，不再依赖泛行业分发
- [x] `深度洞察` 改为“强深度命中 + 专栏兜底”的组合逻辑，不再无差别扩张
- [x] 通过降低多栏目同时命中，削弱首页与栏目页重复感

## 验收标准

- [x] 专题总数达到 12-18 个，其中至少一半不是纯 seed 直出
- [x] 新增专题不出现明显低信号专题名，如单一公司名或泛职业标签专题
- [x] 主题间文章重叠度保持可控，批量抽样不出现明显同质专题
- [x] `行业观察` 中无 `industry` 标签文章占比显著下降
- [x] `院长说` 抽样结果中，大多数文章能直接解释其“院长 / 教授 / 核心师资观点”归属
- [x] 栏目平均命中数下降到更可控范围，并完成首页/栏目页抽样检查
- [x] 补充至少 1 组聚类质量测试或校验脚本，避免后续回退

## 建议执行顺序

1. [x] 先补一份聚类体检脚本，输出专题数量、专题重叠、栏目重叠、误入比例

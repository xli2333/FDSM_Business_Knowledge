# Round46 英文首页数据链路与搜索建议本地化
## 目标

解决英文模式首页仍然出现中文内容和英文主视觉换行不稳的问题。

- 首页 feed、最新文章和搜索建议都要支持 `language=en`
- 英文首页优先引用英文文章标题、摘要、标签、专题和栏目文案
- 对没有英文翻译的数据做英文兜底或过滤，避免英文首页直接露出中文
- 只调整英文分支，不改变中文首页现有结构和内容

## 原子任务

- [x] P0 | 后端本地化 | 新增首页本地化 helper，覆盖 tag / topic / column 的英文显示
- [x] P0 | 首页 feed | 为 `/home/feed` 与 `/articles/latest` 增加 `language` 参数并返回英文数据
- [x] P0 | 搜索建议 | 为 `/suggest` 增加 `language` 参数，英文模式优先返回英文标题和英文标签建议
- [x] P0 | 前端消费 | 首页与首页搜索建议在英文模式下传入 `language=en`
- [x] P0 | 英文布局 | 仅调整英文首页 Hero 的列宽与字号，确保第二行文案保持单行
- [x] P1 | 英文兜底 | 英文模式过滤仍含中文的文章卡片，避免首页直接显示中文内容
- [x] P1 | 验收 | 运行构建、Lint 和后端自检，确认英文首页 feed 与英文建议结果不含中文
- [x] P1 | 回填 | 修正本轮 Todo 编码并按完成情况逐项勾选

## 本轮结论

- 英文首页数据链路已经切到英文 feed，首页首屏、编辑精选、快速入口、专题起读、栏目预览和最新文章都走英文数据或英文兜底
- 英文首页搜索建议已支持英文返回，避免在英文首页输入时弹出中文建议
- 英文 Hero 只对英文分支做了宽度和字号收口，`New ground for change` 不再挤成两行
- 当前首页范围内没有更明确、值得继续开下一轮的提升项，本轮结束

# 配置密钥与数据资产交接清单

生成日期：2026-06-01

## 交接原则

生产配置和数据资产不要混在代码包里交接。代码可以通过 Git 或发布包传递，真实 `.env.production`、数据库、上传文件、Gemini key、CAS 信息、备份凭据必须通过安全渠道交接。

## 环境文件

| 文件 | 当前定位 | 是否可提交 | 是否上服务器 |
| --- | --- | --- | --- |
| `.env.production.example` | 生产模板 | 可以 | 可以 |
| `.env.docker.example` | 本地 Docker 模板 | 可以 | 可选 |
| `.env.production` | 真实生产配置 | 不可以 | 必须，但由服务器安全保存 |
| `.env.docker` | 本地 Docker 配置 | 不可以 | 不需要 |
| `.env` | 本地开发配置 | 不可以 | 不需要 |

本轮本地核对发现：`.env` 和 `.env.docker` 中存在非空 Gemini key。不要把这些值写入文档、截图、日志、发布包或聊天记录；正式交接前需要确认 key 是否继续有效，必要时由 key 管理方轮换或撤销。

服务器上的 `.env.production` 应设置：

```bash
chmod 600 .env.production
```

或在运维要求下使用：

```bash
chmod 640 .env.production
```

## 生产必填配置

### 站点和 CORS

| 变量 | 状态要求 | 说明 |
| --- | --- | --- |
| `APP_ENV` | `production` | 生产硬要求 |
| `APP_PORT` | 例如 `8080` | 前端容器绑定宿主机端口 |
| `SITE_BASE_URL` | 正式 HTTPS 域名 | 不能是 `127.0.0.1` 或占位域名 |
| `ALLOWED_ORIGINS` | 正式 HTTPS origin | 生产不能用 `*` |
| `FDSM_DATA_DIR` | Docker 内为 `/data` | 由 compose 设置 |

### CAS 身份

| 变量 | 状态要求 | 说明 |
| --- | --- | --- |
| `AUTH_BACKEND` | `cas` | 生产主线 |
| `CAS_ENABLED` | `1` | 开启 CAS |
| `CAS_SERVER_URL` | 学校 CAS 地址 | 需要由身份系统确认 |
| `CAS_SERVICE_BASE_URL` | 正式 HTTPS 域名 | 与站点域名一致 |
| `CAS_SERVICE_URL` | `https://域名/api/auth/cas/callback` | 必须加入 CAS 白名单 |
| `CAS_VALIDATE_TIMEOUT_SECONDS` | 建议 `8` | CAS 校验超时 |
| `CAS_SESSION_TTL_SECONDS` | 建议 `28800` | session 有效期 |
| `CAS_SESSION_RETENTION_DAYS` | 建议 `30` | 过期 session 保留 |
| `CAS_ADMIN_EMPLOYEE_NUMBERS` | 待填写 | 管理员工号 |
| `CAS_ADMIN_USERNAMES` | 待填写 | 管理员用户名 |

生产不建议启用旧 Supabase 路径，除非做受控回滚：

```env
ALLOW_LEGACY_SUPABASE_AUTH=0
SUPABASE_URL=
SUPABASE_ANON_KEY=
```

### AI/Gemini

| 变量 | 状态要求 | 说明 |
| --- | --- | --- |
| `GOOGLE_API_KEY` | 至少提供一个可用 key | 单 key |
| `GEMINI_API_KEYS` | 可选，多 key 逗号分隔 | key 轮换 |
| `GEMINI_CHAT_MODEL` | 当前模板为 `gemini-3.0-flash` | 实际运行会做别名映射 |
| `GEMINI_FLASH_MODEL` | 当前模板为 `gemini-3.0-flash` | 用于批处理/摘要等 |
| `RAG_SEARCH_PROVIDER` | `local_chunk` | 当前主路径 |
| `RAG_ENABLE_INLINE_INGESTION` | `1` | 允许 inline 入库 |
| `RAG_CHUNK_EMBEDDINGS_ENABLED` | `1` | 生成 chunk embedding |

交接时需要说明：

- key 由谁申请。
- key 限额是多少。
- 是否允许生产批处理。
- 如果 key 失效，谁负责更换。
- 是否允许记录 Gemini 错误日志给外部告警。

### Redis 和异步任务

| 变量 | 状态要求 | 说明 |
| --- | --- | --- |
| `REDIS_PASSWORD` | 真实强密码 | preflight 强制要求 |
| `REDIS_MAXMEMORY` | 例如 `512mb` | Redis 内存上限 |
| `ASYNC_TASKS_ENABLED` | `1` | 生产开启 |
| `ASYNC_TASK_QUEUE_KEY` | `fdsm:async-tasks` | 队列 key |
| `ASYNC_TASK_DEAD_LETTER_QUEUE_KEY` | `fdsm:async-tasks:dead-letter` | 死信队列 |
| `ASYNC_TASK_POLL_TIMEOUT_SECONDS` | 例如 `5` | worker 拉取超时 |

### Metrics、日志和 Sentry

| 变量 | 状态要求 | 说明 |
| --- | --- | --- |
| `METRICS_ENABLED` | `1` 或明确关闭 | 开启时必须有 token |
| `METRICS_TOKEN` | 真实强 token | 不可为空或占位 |
| `REQUEST_LOG_JSON_ENABLED` | 建议 `1` | 结构化请求日志 |
| `SENTRY_REQUIRED` | 按运维要求 | 如果为 `1` 必须填 DSN |
| `SENTRY_DSN` | 真实 DSN 或明确关闭 | 错误上报 |
| `SENTRY_TRACES_SAMPLE_RATE` | 例如 `0.05` | tracing 采样 |

### 上传大小

| 变量 | 模板默认 | 说明 |
| --- | ---: | --- |
| `MEDIA_AUDIO_UPLOAD_MAX_BYTES` | 536870912 | 512 MB |
| `MEDIA_VIDEO_UPLOAD_MAX_BYTES` | 2147483648 | 2 GB |
| `MEDIA_TEXT_UPLOAD_MAX_BYTES` | 20971520 | 20 MB |
| `MEDIA_IMAGE_UPLOAD_MAX_BYTES` | 20971520 | 20 MB |

注意：前端容器 Nginx `deploy/nginx/default.conf` 和宿主机 HTTPS Nginx 模板 `deploy/nginx/fdsmarticles-https.conf` 当前都是 `client_max_body_size 200m`。如果生产要上传大于 200 MB 的视频，必须同步调整两处 Nginx。

### 备份

| 变量 | 状态要求 | 说明 |
| --- | --- | --- |
| `BACKUP_INTERVAL_SECONDS` | 例如 `86400` | 每日 |
| `BACKUP_RETENTION_DAYS` | 例如 `14` | 本地保留 |
| `BACKUP_COMPRESS` | `1` | 压缩 |
| `BACKUP_OFFSITE_REQUIRED` | `1` | 生产建议必须 |
| `BACKUP_OFFSITE_TARGET` | 真实目标 | rclone remote 或等价 |
| `BACKUP_ENCRYPTION_REQUIRED` | `1` | 生产建议必须 |
| `BACKUP_ENCRYPTION_RECIPIENT` | 真实 GPG recipient | 加密收件人 |
| `BACKUP_OFFSITE_COMMAND` | `rclone` | 异地同步工具 |

交接时必须提供：

- rclone remote 配置方式或对象存储账号。
- GPG 公钥/收件人。
- 恢复权限负责人。
- 备份保留策略。
- 恢复演练记录。

当前仓库的 backup 服务没有 Docker healthcheck，备份失败告警也不在仓库内闭环；生产必须由运维监控或外部任务系统补足。

## 数据库交接

当前数据库：

- 文件：`data/fudan_knowledge_base.db`
- 大小：约 703.8 MB
- 业务表数量：47，已排除 `sqlite_sequence` 等 SQLite 内部表
- 内容量级：2146 篇文章，2410 个标签，16 个专题，22 个媒体项，8900 个 RAG chunks，8900 个 embeddings。
- 最新本地交接压缩备份：`backups/fudan_knowledge_base.handoff_20260602-142839.db.gz`，约 223.3 MB，`PRAGMA quick_check` 结果为 `ok`。

服务器上线前检查：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml exec backend-web python -m backend.scripts.db_diagnostics
```

也可直接 SQLite 检查：

```sql
PRAGMA quick_check;
PRAGMA journal_mode;
SELECT COUNT(*) FROM articles;
SELECT COUNT(*) FROM article_chunks;
SELECT COUNT(*) FROM article_chunk_embeddings;
```

## FAISS 交接

路径：

```text
data/faiss_index_business/index.faiss
data/faiss_index_business/index.pkl
```

交接要求：

- 明确这是本地已有索引，不等于 ARM64 已验证索引。
- 在华为 ARM64 服务器上必须重新验证。
- 验证失败则在 ARM64 环境重建。
- 验证结果要写入上线记录。

## 上传和媒体资产交接

`data/uploads/` 当前包含：

- 编辑台封面图。
- 媒体音频封面。
- 媒体视频封面。
- 媒体音频原文件。
- 媒体视频原文件。
- 转录稿。
- 脚本/文稿。

生产 Nginx 路径设计：

- `/editorial-uploads/`：公开缓存。
- `/media-uploads/audio/cover/`、`/media-uploads/video/cover/`：公开缓存。
- `/media-uploads/audio/media/`、`/media-uploads/video/media/`：internal，必须走后端鉴权。
- `/audio-files/`：internal。

交接时必须验证上传目录权限和 Nginx alias 一致。

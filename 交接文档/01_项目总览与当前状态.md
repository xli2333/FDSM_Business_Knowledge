# 项目总览与当前状态

生成日期：2026-06-01

## 项目定位

`fdsmarticles` 是一个面向复旦管院商业知识内容的业务系统。它不是单纯的前端页面，也不是只展示文章的静态站点，而是一个内容知识平台，包含：

- 内容浏览：文章、首页 feed、搜索、栏目、标签、专题、机构、时光机。
- AI/RAG：公开知识库检索、AI 对话助手、文章上下文、用户个人知识主题。
- 编辑 CMS：草稿、发布、重编、AI 自动排版、AI 摘要、翻译、专题挂载、公众号排版输出。
- 音视频：音频/视频内容、封面、转录稿、脚本、流式播放、付费权限控制。
- 用户系统：CAS 生产身份接入、旧 Supabase 回滚隔离、本地开发调试身份。
- 会员和商业化：免费/付费会员、会员权限、商业落地页、线索管理、订单/订阅骨架。
- 后台运营：内容运营后台、RAG 后台、会员后台、分析页、管理员工作台。
- 部署运维：Docker 生产栈、Nginx、安全头、限流、备份、恢复、验收脚本、蓝绿发布、回滚脚本。

- 前后端分离架构已经完成。

- CAS 接入代码、生产 debug auth 禁用逻辑、CORS 生产约束、metrics token、结构化日志、Sentry 接入口已经存在。

## 当前不能直接上线的原因

### 1. 当前 `.env.production` 不是正式生产配置

当前工作区 `.env.production` 只是本地占位/联调口径，不可生产使用。它仍包含本地访问口径，例如 `SITE_BASE_URL`、`ALLOWED_ORIGINS` 为 `127.0.0.1` 类地址；`CAS_ENABLED=0`，CAS、Gemini、Redis 密码、metrics token、Sentry、备份加密和异地目标等生产必填项未完整落地。与 `.env.production.example` 相比，当前真实文件缺少 57 个键，关键缺项包括 `REDIS_PASSWORD`、`METRICS_TOKEN`、`SENTRY_DSN`、`MEDIA_*_UPLOAD_MAX_BYTES`、`ASYNC_TASK_DEAD_LETTER_QUEUE_KEY`、容器资源限制键、备份加密/异地相关键等。

正式上线必须按 `.env.production.example` 重新生成服务器专用 `.env.production`，并由运维或安全负责人保管。

### 2. 华为/麒麟 ARM64 需要实机构建和验证

如果服务器是华为 Kunpeng ARM64，不能默认使用本地 Windows/x86 环境生成的镜像或 FAISS 索引。后端镜像依赖 Python、Node、`faiss-cpu`、`numpy`、`cryptography`、`google-genai` 等依赖，必须在 ARM64 上重新构建并验证。

FAISS 索引尤其需要注意：当前 `data/faiss_index_business/index.faiss` 和 `index.pkl` 是本地已有产物，跨平台直接加载必须以 ARM64 容器实测结果为准。当前业务 RAG 主路径是 `local_chunk`，即 `article_chunks` + `article_chunk_embeddings`；FAISS 不可用时，`local_chunk` RAG 仍可能工作，但旧/辅助 FAISS 能力会缺失。因此“页面能打开”不代表“全部检索能力正常”，上线验收必须同时检查 `local_chunk` 命中和 FAISS 显式加载。

### 3. CAS 和正式域名不是代码能单方面完成

生产登录依赖学校 CAS。必须完成：

- 正式 HTTPS 域名。
- CAS 服务端白名单。
- CAS 回调地址注册。
- 管理员工号/用户名配置。
- 登录、回调、退出、session 过期的联调。

这些工作依赖校内身份系统和服务器域名，不是本地代码直接决定。

### 4. Gemini key 和网络连通性决定 AI 功能是否可用

编辑台 AI、摘要、翻译、RAG 对话等依赖 Gemini key。此前全量测试中已经出现过本地 Gemini key 无效导致 AI 发布相关测试失败的情况。上线前必须使用生产 key 做真实调用验证，并准备错误率监控和 key 轮换策略。

### 5. 备份、恢复和监控还需要生产侧闭环

项目已经有备份/恢复脚本和 backup 服务，但正式上线前必须配置真实异地目标、GPG 加密收件人、保留周期、恢复演练和告警。`docker-compose.prod.yml` 中 backup 服务当前没有 healthcheck，备份失败告警也未在仓库内落地；没有恢复演练和失败告警的备份不能算生产可用。

### 6. 当前测试状态不是“全绿”

本次交接复核检查结果：

- `python -m compileall backend deploy`：通过。
- `npm --prefix frontend run build`：通过；同时提示 Browserslist/caniuse-lite 数据偏旧。
- `python -m pytest backend/tests/test_media_stream_auth.py backend/tests/test_metrics_routes.py -q`：5 passed。
- `docker compose --env-file .env.production.example -f docker-compose.prod.yml config`：通过。
- `docker compose --env-file .env.docker.example config`：通过。
- `npm --prefix frontend run lint`：失败，输出为 `50 problems (49 errors, 1 warning)`，需要整理后作为上线门槛。
- `docker compose --env-file .env.production -f docker-compose.prod.yml config`：失败，直接原因是当前真实 `.env.production` 缺 `REDIS_PASSWORD`；同时该文件还缺多个生产模板键，不能作为正式生产配置。
- `python -m pytest backend/tests -q --maxfail=1 --durations=20`：此前曾因本地 Gemini API key 无效导致编辑发布链路测试失败，正式上线前仍需用可用 key 或隔离外部 AI 调用后重跑。
- 本地没有完成真实 Docker image build 验证；华为 ARM64 服务器上还必须重新构建验证。

因此，项目工程资产较完整，但上线前仍需要一轮生产环境级验收。

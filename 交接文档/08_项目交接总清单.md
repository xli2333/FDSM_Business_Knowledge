# 项目交接总清单

生成日期：2026-06-01

本文是 `fdsmarticles` 项目的总交接文件，用于让接手方快速知道：项目现在是什么、代码和数据在哪里、哪些内容必须交接、哪些内容还不能直接上线、接下来应该按什么顺序处理。

## 1. 项目当前定位

`fdsmarticles` 是一个商业知识内容平台，当前不是单纯静态页面，也不是只有前端的展示站点。它包含：

- 前端 React/Vite 单页应用。
- 后端 FastAPI API 服务。
- SQLite 主数据库。
- 公开内容浏览、搜索、栏目、标签、专题、机构页面。
- AI/RAG 检索和 AI 对话能力。
- 编辑 CMS、草稿、发布、重编、AI 摘要、翻译、自动排版。
- 公众号排版相关 Node 运行时。
- 音频/视频媒体模块和付费媒体鉴权。
- 用户、会员、CAS 登录、管理员权限。
- 商业线索、计费计划、订单/订阅骨架。
- Docker Compose 生产栈、Nginx、Redis、worker、backup 服务。
- 预检、验收、备份、恢复、蓝绿、回滚脚本。

当前项目具备进入服务器部署联调的条件，但不建议在未完成生产配置、数据迁移、ARM64 验证、CAS 联调、备份恢复和监控告警前直接对外开放。

## 5. 运行架构交接

生产 Compose 主要服务：

| 服务 | 作用 | 交接重点 |

| `backend-web` | FastAPI Web API | 依赖 `.env.production`、`data/`、Redis、Gemini、CAS |

默认生产访问路径：

- 宿主机 Nginx/HTTPS 接收外部流量。
- 转发到本机 `APP_PORT`，默认示例为 `8080`。
- 前端容器 Nginx 提供静态页面和 `/api` 反代。
- 后端通过 `/api/*` 对外提供业务接口。

## 6. 后端模块交接

关键入口和职责：

- `backend/main.py`：创建 FastAPI app，注册中间件、health、ready、metrics 和业务 router。
- `backend/config.py`：统一读取生产配置，包含数据路径、Gemini、RAG、Redis、CAS、metrics、Sentry、上传限制等。
- `backend/database.py`：SQLite 连接、WAL、ready 检查、运行时表创建。
- `backend/worker.py`：异步任务 worker。
- `backend/worker_healthcheck.py`：worker 健康检查。
- `backend/routers/`：业务 API 路由。
- `backend/services/`：业务服务层。
- `backend/scripts/`：数据构建、迁移、诊断、批处理、运维脚本。
- `backend/tests/`：后端测试。
- `backend/wechat_runtime/`：公众号排版 Node 运行时。

必须说明给接手方：

- 当前主数据库是 SQLite，不是 PostgreSQL。
- `DATABASE_BACKEND`、`DATABASE_URL` 有配置项，但不等于已经完整支持 PostgreSQL adapter。
- 后端大量 SQL 当前按 SQLite 直接执行，不能只填 `DATABASE_URL` 就迁移数据库。
- `backend-worker` 当前映射的编辑类异步任务包括 `editorial.auto_format`、`editorial.auto_summary`、`editorial.auto_translate`，不要理解为所有 AI/RAG 都已进入 worker。

## 12. ARM64/华为麒麟交接

如果服务器是华为 Kunpeng/ARM64 或麒麟 ARM64：

- 必须确认 `uname -m` 为 `aarch64`。
- 不能直接使用本地 Windows/x86 机器构建的 amd64 镜像。
- 必须在 ARM64 服务器上 `docker compose build`，或用 buildx 构建 `linux/arm64` 镜像。
- 必须验证 Python 原生依赖：`faiss-cpu==1.13.2`、`numpy`、`cryptography`、`pillow` 等。
- 必须验证 Node 原生依赖：`backend/wechat_runtime` 中的 `sharp`。
- 必须显式验证 `data/faiss_index_business`。
- 必须做压测和资源监控，不能只看容器能启动。

特别注意：

- `backend/wechat_runtime/package.json` 依赖 `@google/genai`、`sharp`、`undici`。
- `sharp` 是原生依赖，`package-lock.json` 中包含 linux arm64 包，但仍要以 ARM64 容器构建结果为准。

## 15. 备份、恢复和回滚交接

备份脚本：

- `deploy/backup_sqlite.py`
- `deploy/restore_sqlite_backup.py`

一次性备份命令：

```bash
docker compose --env-file .env.production -f docker-compose.prod.yml exec backup python /ops/backup_sqlite.py
```

循环备份由 compose 中 backup 服务使用：

```bash
python /ops/backup_sqlite.py --loop
```

交接重点：

- 生产必须配置真实异地备份目标。
- 生产建议开启 GPG 加密，并配置真实收件人。
- 当前 backup 服务没有 Docker healthcheck。
- 备份失败告警不在仓库内闭环，必须由外部监控或任务系统补足。
- 只有备份文件存在不代表可恢复，必须做恢复演练。

恢复演练要求：

- 不直接覆盖生产库。
- 先恢复到临时目录或临时 SQLite 文件。
- 执行 `PRAGMA quick_check`。
- 抽样查 `articles`、`media_items`、`article_chunks`。
- 记录恢复耗时。

回滚脚本：

- `deploy/rollback_release.sh`
- `deploy/blue_green_deploy.sh`

如果涉及数据库结构变更，不建议两个版本并行共享同一个 SQLite 文件，需要维护窗口、完整备份和专门迁移方案。

- 正式服务器架构和 Docker 环境未实机验证。
- 华为/麒麟/ARM64 需要重新构建镜像。

# 系统模块与代码职责地图

生成日期：2026-06-01

## 总体架构

```text
浏览器
  -> 宿主机 Nginx HTTPS
  -> frontend 容器 Nginx
      -> React/Vite 静态页面
      -> /api/* 反代 backend-web:8000
  -> backend-web FastAPI
      -> SQLite data/fudan_knowledge_base.db
      -> Redis 队列
      -> Gemini API
      -> data/uploads, data/audio, data/faiss_index_business
  -> backend-worker
      -> Redis 异步任务
      -> SQLite / Gemini / 编辑类 AI 异步任务
  -> backup 容器
      -> SQLite 备份、压缩、加密、异地同步
```

## 后端入口职责

`backend/main.py` 负责：

- 创建 FastAPI app。
- 启动时创建上传/音频目录。
- 确保数据库 ready 和运行时表 ready。
- 清理过期 CAS session。
- 同步本地音频库。
- 注册 CORS。
- 生产环境拒绝 debug auth header。
- 输出结构化请求日志。
- 暴露 `/api/health`、`/api/ready`、`/api/metrics`。
- 注册所有业务 router。

生产镜像默认命令：

```bash
gunicorn backend.main:app -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
```

## 配置职责

`backend/config.py` 统一读取环境变量，关键配置包括：

- `APP_ENV`、`IS_PRODUCTION`。
- `FDSM_DATA_DIR`、SQLite、FAISS、uploads、audio 路径。
- Gemini key 和模型。
- RAG provider、chunk、embedding 开关。
- Redis 队列配置。
- 数据库写重试、慢查询、观测配置。
- metrics、Sentry、结构化日志。
- CAS、Supabase 回滚隔离、管理员身份。
- 会员、支付 provider。
- 上传大小限制。

生产环境必须避免隐式默认值造成误上线，关键值应全部在 `.env.production` 明确写出。

## 数据库职责

`backend/database.py` 当前以 SQLite 为主：

- 通过 `sqlite3` 直连 `DATA_DIR/fudan_knowledge_base.db`。
- 启用 WAL、busy timeout、foreign keys。
- 提供写事务重试。
- 提供 ready 检查。
- 缺核心库或重建时调用 `backend/scripts/build_business_db.py`。
- 幂等创建/迁移运行时表。

当前 `DATABASE_BACKEND`、`DATABASE_URL` 有配置项，但并不等于已经完整支持 PostgreSQL adapter。若要切换 PostgreSQL，需要专门评估 SQL、事务、迁移和服务层访问方式。

核心表类别：

- 内容：`articles`、`tags`、`article_tags`、`columns`、`article_columns`、`topics`、`topic_articles`。
- 用户：`visitor_profiles`、`business_users`、`auth_sessions`、`user_memberships`。
- 行为：`article_view_events`、`article_reactions`、`user_saved_articles`、`user_follows`。
- AI/RAG：`article_chunks`、`article_chunk_embeddings`、`ingestion_jobs`、`retrieval_events`。
- CMS：`editorial_articles`、`editorial_article_topics`、`article_versions`、`article_ai_outputs`。
- 媒体：`media_items`、`media_drafts`。
- 商业化：`billing_plans`、`billing_orders`、`billing_subscriptions`、`billing_checkout_intents`、`demo_requests`。
- 运维：`async_tasks`、`meta`。

## AI/RAG 模块

AI 相关服务：

- `backend/services/ai_service.py`：Gemini 调用、搜索重排、摘要/翻译/编辑类 AI 能力。
- `backend/services/knowledge_retrieval_service.py`：当前 `RAG_SEARCH_PROVIDER=local_chunk` 主路径，主要基于 `article_chunks`、`article_chunk_embeddings` 做检索。
- `backend/services/rag_engine.py`：旧/辅助 FAISS 业务知识检索逻辑，读取 `data/faiss_index_business`。
- `backend/services/knowledge_ingestion_service.py`：公开或用户知识入库。
- `backend/services/knowledge_chunking_service.py`：文本切块。
- `backend/services/knowledge_embedding_service.py`：embedding 生成。
- `backend/services/rag_admin_service.py`：后台 RAG 资产和任务可视化。

需要注意：

- 当前主检索配置是 `local_chunk`，FAISS 不是唯一 RAG 路径；FAISS 加载失败时项目可能降级，不一定导致整个应用启动失败。
- 因此上线验收不能只看 `/api/ready`，也不能只验证其中一个路径：要验证 `local_chunk` 检索命中质量，也要显式验证 `data/faiss_index_business` 在 ARM64 容器内可加载或已重建。
- Gemini key 不可用时，内容浏览可以继续，但 AI 摘要、翻译、编辑、对话会受影响。
- `backend-worker` 当前消费 Redis 异步任务，已映射的编辑类任务包括 `editorial.auto_format`、`editorial.auto_summary`、`editorial.auto_translate`。不要把它理解成已经覆盖所有 AI/RAG 工作负载的通用 worker。

能力范围：

- 音频/视频列表和详情。
- 管理员上传媒体、封面、脚本、转录稿。
- 媒体内容发布与草稿管理。
- 付费内容权限控制。
- Nginx `internal` alias + 后端鉴权 + `X-Accel-Redirect` 控制媒体原文件。

上线验收重点：

- 未登录或无权限访问付费媒体应返回 401/403。
- 有权限访问应支持 200/206 流式播放。
- Nginx 和 backend 日志不能泄漏 token。
- `client_max_body_size` 与后端上传大小限制要一致评估，目前前端容器 Nginx 和宿主机 HTTPS Nginx 模板都是 200m，而后端视频上传默认可到 2GB；如果生产需要大视频上传，两处 Nginx 配置都必须调整。

## 前端认证链路

- `AuthProvider.jsx` 请求 `/api/auth/status`，维护 `user`、`session`、`membership`、`businessProfile`、`roleHomePath`、`authMode`。
- `ProtectedRoute.jsx` 根据 `isAuthenticated`、`isAdmin`、`isPaidMember`、`canUseAiAssistant` 控制跳转。
- `ProtectedRoute.jsx` 支持 `requirePaid`，但当前 `App.jsx` 中未看到实际路由使用该参数；会员/付费权限还需要结合后端接口和内容权限做专项验收。
- `LoginPage.jsx` 根据后端返回的 `auth_mode === 'cas'` 显示 CAS 登录。
- `CasCallbackPage.jsx` 从 URL hash 读取 token，保存到 `localStorage` 的 `fdsm-cas-token`。
- `api/index.js` 后续自动发送 `Authorization: Bearer <token>`。
- 本地开发时 `fdsm-debug-auth` 可生成 debug auth header；`api/index.js` 在存在 CAS token 时优先发送 CAS Bearer token。生产 Docker build arg 已设置 `VITE_ENABLE_DEBUG_AUTH=0`，后端也会拒绝 debug header。

## 部署和运行模块

生产 Compose 服务：

| 服务 | 职责 |
| --- | --- |
| `backend-web` | FastAPI Web 服务 |
| `backend-worker` | Redis 异步任务消费 |
| `backend-housekeeping` | 周期清理 CAS session 并写 heartbeat |
| `frontend` | Nginx 托管前端静态文件并反代 `/api` |
| `redis` | 异步任务队列和缓存 |
| `backup` | SQLite 备份、压缩、加密、异地同步 |

健康检查：

- backend-web：`/api/ready`。
- backend-worker：`python -m backend.worker_healthcheck`。
- backend-housekeeping：heartbeat 文件。
- frontend：`/healthz`。
- redis：`redis-cli ping`。

# 环境变量与密钥交接说明

生成日期：2026-06-02

本文说明哪些环境配置可以随 GitHub 交接，哪些必须通过安全渠道单独交接。原则是：交接信息要完整，但真实 key、密码、token 不进入仓库、普通压缩包或聊天记录。

## 1. 当前结论

可以上传到 GitHub：

- `.env.docker.example`
- `.env.production.example`
- 本文档中的脱敏说明

不上传到 GitHub：

- `.env`
- `.env.docker`
- `.env.production`
- 任何真实 Gemini/Google API key
- 任何真实 Redis 密码、CAS 凭据、Sentry DSN、metrics token、备份凭据

当前 `.gitignore` 已忽略真实环境文件：

```text
.env
.env.*
```

并明确允许模板文件：

```text
!.env.docker.example
!.env.production.example
```

## 2. 本地 `.env`

当前 `.env` 只用于本地开发脚本或非 Docker 场景，包含：

| 变量 | 当前状态 | 是否可公开 | 说明 |
| --- | --- | --- | --- |
| `GOOGLE_API_KEY` | 已设置 | 否 | 真实 Google/Gemini key，必须私下交接或重新申请 |
| `GEMINI_API_KEYS` | 已设置 | 否 | 真实 Gemini key 列表，必须私下交接或重新申请 |
| `GEMINI_CHAT_MODEL` | `gemini-3-flash` | 是 | 模型名，不涉密 |
| `GEMINI_EMBEDDING_MODEL` | `models/gemini-embedding-001` | 是 | embedding 模型名，不涉密 |

交接建议：不要把当前 `.env` 文件发给同事。让同事按需新建 `.env`，只填自己的 key。

## 3. 本地 Docker `.env.docker`

当前 `.env.docker` 用于本机 Docker 开发运行，核心非敏感值如下：

```env
APP_ENV=development
APP_PORT=8080
SITE_BASE_URL=http://127.0.0.1:8080
ALLOWED_ORIGINS=http://127.0.0.1:8080
FDSM_DATA_DIR=/data
DATABASE_BACKEND=sqlite
DATABASE_URL=
DEV_AUTH_ENABLED=1
VITE_ENABLE_DEBUG_AUTH=1
AUTH_BACKEND=auto
CAS_ENABLED=0
RAG_SEARCH_PROVIDER=local_chunk
RAG_ENABLE_INLINE_INGESTION=1
RAG_CHUNK_EMBEDDINGS_ENABLED=1
ASYNC_TASKS_ENABLED=1
ASYNC_TASK_QUEUE_KEY=fdsm:async-tasks
GEMINI_CHAT_MODEL=gemini-3.0-flash
GEMINI_FLASH_MODEL=gemini-3.0-flash
GEMINI_EMBEDDING_MODEL=models/gemini-embedding-001
```

需要私下补齐或由同事自行申请：

```env
GOOGLE_API_KEY=<真实 key>
GEMINI_API_KEYS=<真实 key，可多个逗号分隔>
```

同事本地复现建议：

```powershell
Copy-Item .env.docker.example .env.docker
```

然后根据上面的本地开发口径调整 `.env.docker`，并填入自己的 Gemini key。

## 4. 生产 `.env.production`

当前本地 `.env.production` 更像生产模拟配置，不是最终可上线配置。它没有填真实生产 Gemini key，也没有填正式域名或 CAS 白名单信息。

当前非敏感值包括：

```env
APP_ENV=production
APP_PORT=18081
SITE_BASE_URL=http://127.0.0.1:18081
ALLOWED_ORIGINS=http://127.0.0.1:18081
FDSM_DATA_DIR=/data
DATABASE_BACKEND=sqlite
DEV_AUTH_ENABLED=0
VITE_ENABLE_DEBUG_AUTH=0
AUTH_BACKEND=auto
CAS_ENABLED=0
RAG_SEARCH_PROVIDER=local_chunk
RAG_ENABLE_INLINE_INGESTION=1
RAG_CHUNK_EMBEDDINGS_ENABLED=1
ASYNC_TASKS_ENABLED=1
ASYNC_TASK_QUEUE_KEY=fdsm:async-tasks
REDIS_MAXMEMORY=512mb
BACKUP_INTERVAL_SECONDS=86400
BACKUP_RETENTION_DAYS=14
```

正式上线前必须由接手方补齐：

```env
SITE_BASE_URL=https://正式域名
ALLOWED_ORIGINS=https://正式域名
AUTH_BACKEND=cas
CAS_ENABLED=1
CAS_SERVER_URL=<学校 CAS 地址>
CAS_SERVICE_BASE_URL=https://正式域名
CAS_SERVICE_URL=https://正式域名/api/auth/cas/callback
CAS_ADMIN_EMPLOYEE_NUMBERS=<管理员工号>
CAS_ADMIN_USERNAMES=<管理员用户名>
GOOGLE_API_KEY=<生产 Gemini key>
GEMINI_API_KEYS=<生产 Gemini key 列表>
REDIS_PASSWORD=<强密码>
METRICS_TOKEN=<强 token>
SENTRY_DSN=<如启用 Sentry>
BACKUP_OFFSITE_TARGET=<异地备份目标>
BACKUP_ENCRYPTION_RECIPIENT=<备份加密收件人>
```

生产环境文件应放在服务器本地，并设置权限：

```bash
chmod 600 .env.production
```

## 5. 给同事的最小配置路径

本地开发：

```powershell
git clone https://github.com/xli2333/FDSM_Business_Knowledge.git
cd FDSM_Business_Knowledge
Copy-Item .env.docker.example .env.docker
```

然后：

- 解压单独交接的数据包，把 `data/` 放到仓库根目录。
- 在 `.env.docker` 中填入同事自己的 `GOOGLE_API_KEY` 或 `GEMINI_API_KEYS`。
- 执行 `docker compose --env-file .env.docker up -d --build`。

生产部署：

```bash
cp .env.production.example .env.production
```

然后按正式域名、CAS、Gemini、Redis、metrics、备份策略补齐真实值。

## 6. 安全边界

以下内容即使为了交接，也不要公开上传：

- Gemini/Google API key
- Redis 密码
- CAS 管理员名单和生产 CAS 回调白名单材料
- Sentry DSN
- metrics token
- rclone、对象存储、GPG 备份凭据
- 任何个人账号密码

如果接手同事必须使用现有 key，应通过学校内部密钥系统、加密压缩包、线下口令、企业密码管理器或同等安全渠道交接，并记录 key 负责人和轮换计划。
